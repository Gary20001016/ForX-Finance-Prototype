# 外部机翻与人工审核流程设计

**日期：** 2026-07-13
**范围：** 消息模板多语言生产、翻译审核、消息发布门禁，以及对应管理端原型。

## 1. 目标

为交易所消息中心建立一条可追踪、可重试、可审计的多语言内容生产流程：操作者维护默认语言源文案并选择目标语言，平台通过外部异步任务接口发起机器翻译，翻译结果返回后由人工逐语言审核，所有目标语言均审核通过后才允许进入业务审核和发布。

本次只修改 PRD 和前端原型，不实现真实后台接口或外部翻译服务。

## 2. 核心原则

1. 默认语言内容由操作者直接维护，不进入机器翻译。
2. 一个模板版本对应一个翻译批次，每个目标语言对应一个翻译子任务。
3. 前端只调用消息中心后台，不直接访问外部翻译服务。
4. 外部翻译任务采用异步模式：创建任务后通过完成回调返回结果，后台主动查询作为超时和回调丢失的兜底。
5. 翻译成功不等于可发布；每个目标语言必须完成人工审核。
6. 任一目标语言处于失败、待审核或已驳回状态时，整个模板版本禁止发布。
7. 已审核内容发生修改后，该语言审核结果自动失效，必须重新审核。

## 3. 用户流程

### 3.1 创建翻译批次

操作者完成默认语言的标题、摘要、正文、风险提示、按钮文案和跳转链接，选择一个或多个目标语言并点击“提交机翻”。前端先校验源文案及变量格式，再向平台后台提交翻译批次。

后台为批次生成 `translation_batch_id`，并为每种目标语言创建子任务。调用外部接口后保存其 `external_task_id`。前端展示批次进度和各语言状态。

### 3.2 接收翻译结果

外部翻译服务通过回调返回任务状态与翻译结果。后台完成签名、时间戳、幂等和任务归属校验后保存结果。若在配置时间内未收到回调，后台主动查询外部任务。

失败语言可单独重试，成功语言不重复翻译。重试会生成新的尝试编号，并保留旧结果和错误信息用于审计。

### 3.3 人工审核

翻译成功的语言进入“待人工审核”。审核人员在对照界面查看：

- 默认语言源文案；
- 机器翻译文案；
- 标准变量及其保留情况；
- 风险用语、数字、币种、交易对和链接一致性检查；
- 该语言的机翻任务编号、完成时间和审核记录。

审核人员可以直接修订机翻结果并通过，或填写原因后驳回重翻。修订内容与原机翻结果分别保存。创建人不能审核自己提交的翻译。

### 3.4 发布门禁

当默认语言内容有效，且所有目标语言均为“审核通过”时，翻译批次变为“全部审核通过”，模板版本才可提交后续业务审核。业务审核继续检查受众、风险等级、有效期、发送策略和合规要求。

模板列表和新建消息任务均展示语言就绪状态。未通过翻译门禁的模板版本不可被消息任务选用。

## 4. 状态模型

### 4.1 翻译批次状态

`未提交 → 机翻处理中 → 待人工审核 → 全部审核通过`

异常或分支状态：`部分失败`、`审核被驳回`、`已取消`。

批次状态按所有目标语言子任务聚合计算，不由前端手工修改。

### 4.2 语言子任务状态

`未提交 → 排队中 → 翻译中 → 待人工审核 → 审核通过`

异常或分支状态：`翻译失败`、`审核驳回`、`已取消`。

`审核驳回`可重新提交机翻；`翻译失败`可单独重试。审核通过后的文案如被修改，状态回到`待人工审核`。

## 5. 数据与接口字段

### 5.1 翻译批次

- `translation_batch_id`：平台翻译批次 ID；
- `object_type`：对象类型，第一期为 `message_template_version`；
- `object_id`、`object_version`：模板及版本；
- `source_locale`：默认语言；
- `target_locales`：目标语言列表；
- `status`：聚合状态；
- `total_count`、`completed_count`、`approved_count`、`failed_count`；
- `created_by`、`created_at`、`updated_at`。

### 5.2 语言翻译子任务

- `translation_item_id`、`translation_batch_id`；
- `target_locale`；
- `external_task_id`：外部任务 ID；
- `attempt_no`：重试次数；
- `status`；
- `source_content_hash`：源文案版本校验；
- `machine_title`、`machine_summary`、`machine_body`、`machine_risk_copy`、`machine_button_text`；
- `reviewed_title`、`reviewed_summary`、`reviewed_body`、`reviewed_risk_copy`、`reviewed_button_text`；
- `error_code`、`error_message`；
- `submitted_at`、`translated_at`、`reviewed_at`；
- `reviewer_id`、`review_result`、`review_comment`。

### 5.3 外部任务创建接口

平台后台向外部服务提交：平台请求 ID、源语言、目标语言、标题、摘要、正文、风险提示、按钮文案、变量白名单、回调地址和内容哈希。外部服务同步返回是否受理及 `external_task_id`，不要求同步返回翻译内容。

### 5.4 回调与查询兜底

回调内容包括外部任务 ID、状态、翻译结果、错误码和完成时间。后台必须校验签名、时间戳、防重放信息与内容哈希，并对重复回调保持幂等。

超过配置时间未完成时，后台主动查询。查询与回调结果冲突时，以外部服务最终状态及后台最新有效时间为准，并记录审计日志。

## 6. 前端设计

### 6.1 消息模板页

模板列表增加“翻译进度”列，显示“3/5 已审核”“机翻中”“部分失败”等聚合状态。模板预览抽屉增加“多语言生产”区域，展示默认语言、目标语言、翻译批次 ID 和语言状态矩阵。

提供以下原型操作：创建多语言版本、提交机翻、查看外部任务、重试失败语言、进入人工审核。发布按钮仅在全部语言审核通过时可用，并明确展示禁用原因。

### 6.2 翻译审核

审核中心增加“翻译审核”页签。审核抽屉采用左右对照布局：左侧默认语言源文案，右侧可编辑的机翻结果。顶部展示目标语言、任务 ID、变量检查和内容风险检查，底部提供“驳回重翻”和“修订并通过”。

### 6.3 新建消息任务

模板选择器仅显示翻译审核已全部通过的模板版本。所选模板区域展示默认语言、可用语言和“翻译审核通过”标识；若尝试使用未就绪版本，则显示发布门禁说明。

### 6.4 交互反馈

- 外部任务排队或翻译中：展示进度，不允许人工审核；
- 翻译失败：展示错误码、错误摘要、重试次数和单语言重试入口；
- 待人工审核：突出待办数量；
- 审核驳回：必须展示原因，并允许重新提交该语言；
- 源文案变化：提示现有翻译和审核已失效，需要重新发起流程。

## 7. 权限与审计

- 内容编辑：维护源文案、选择语言、提交机翻、查看结果；
- 翻译审核：编辑机翻结果、通过、驳回；
- 业务审核：仅在翻译门禁通过后审核业务配置；
- 管理员：配置外部服务、超时、查询间隔和重试上限；
- 审计员：查看所有任务、回调、修改和审核记录。

必须审计提交机翻、外部任务 ID 绑定、回调验签结果、主动查询、失败重试、人工修改、审核结论、源文案变更导致的审核失效及最终发布。

## 8. 异常处理

- 外部接口未受理：语言子任务标记为翻译失败并保留错误码；
- 回调超时：触发主动查询，超过最大等待时间后标记失败；
- 部分语言失败：成功语言保留，失败语言单独重试，整体禁止发布；
- 变量被翻译或丢失：自动检查不通过，禁止进入人工通过操作；
- 源文案已变化：通过内容哈希识别，丢弃过期回调并重新发起翻译；
- 重复回调：幂等处理，不覆盖更新的审核结果；
- 外部服务不可用：显示服务异常，允许稍后重试，不提供绕过机翻审核门禁的发布路径。

## 9. 前端测试与验收

1. 模板页能展示翻译批次和逐语言状态。
2. 翻译失败语言具有单独重试入口，成功语言不受影响。
3. 审核中心具有翻译审核页签和源文案/机翻结果对照。
4. 审核人员可模拟修订并通过或填写原因后驳回。
5. 任一语言未审核通过时发布入口禁用并显示原因。
6. 全部目标语言审核通过后显示可提交业务审核状态。
7. 新建消息任务只能选取翻译审核全部通过的模板版本。
8. PRD 的流程、字段、状态、权限、异常和验收标准与前端文案一致。

## 10. 非本期范围

- 真实外部翻译接口联调；
- 翻译质量模型选择、术语库训练和计费结算；
- 自动跳过人工审核；
- 本多语言子项目不单独实现原生 App 页面和 Push 网关；总产品一期仍必须完成 App Push，且本流程产出的多语言模板必须可直接用于正式 Push 发送；
- 多供应商翻译路由和自动故障切换。
