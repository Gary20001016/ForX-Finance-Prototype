# 人工消息已发布模板锁定设计

## 目标

人工消息模板通过业务审核并进入“已发布”状态后，不允许再修改模板内容或基础配置，保证审核对象、发布内容和审计记录保持一致。

## 锁定范围

- 锁定对象：`status === "已发布"`，且 `usageScope` 为 `manual` 或 `shared` 的模板。
- 不锁定对象：`usageScope === "event"` 的纯事件模板。
- 不以多语言状态作为锁定条件。“全部审核通过”但尚未完成业务审核的模板仍可编辑；编辑后沿用现有逻辑，重置为草稿并重新进入翻译和审核流程。
- 已发布模板仍可查看模板信息、多语言结果、使用任务和审核结果。
- 本次不新增“创建新版本”“复制模板”或“解除锁定”能力。

## 前端交互

### 模板列表

- 已锁定模板不再显示不可操作的“编辑”按钮，改为“查看详情”。
- “查看详情”打开完整只读抽屉，操作者可以查看模板基础信息、变量和各渠道内容，但不能提交任何修改。
- “多语言流程”和“使用任务”保持可用。

### 多语言流程

- 已锁定模板的“编辑源文案”按钮禁用，并显示同一锁定提示。
- 已有翻译结果、审核进度和业务审核信息保持只读可见。
- 不允许从多语言流程绕过列表锁定进入模板编辑器。

### 编辑抽屉

- 已锁定模板复用同一抽屉容器，但渲染专用只读详情，不渲染编辑表单和保存、提交机翻操作。
- 只读详情展示模板 ID、编码、名称、版本、状态、分类、性质、风险、适用场景、所有者、渠道、默认语言、语言覆盖、模板变量和更新时间。
- 只读详情使用现有消息预览组件展示 Web 站内信、App 站内信与 App Push 的真实内容效果。
- 页面入口和抽屉内部都应用锁定判断，避免残留前端状态进入可编辑表单。

## 数据层兜底

- 提供统一判断函数，识别“已发布的人工专用或通用模板”。
- `updateTemplate` 在写入前再次执行判断；命中锁定规则时抛出明确错误，不改变模板数据。
- UI 与数据层共用同一判断规则，避免两个入口出现口径差异。
- 新建模板和纯事件模板的更新逻辑保持不变。

## 错误处理

- 正常页面入口通过“查看详情”进入只读抽屉，抽屉顶部提示“业务审核已通过，已发布模板不可修改”。
- 非正常调用命中数据层保护时返回：“业务审核已通过，已发布的人工消息模板不可修改”。
- 更新失败时不得清空翻译批次、提升版本号或将状态重置为草稿。

## 测试范围

- 人工专用模板已发布时，数据层更新被拒绝且原数据不变。
- 通用模板已发布时，数据层更新被拒绝。
- 纯事件模板已发布时，仍沿用现有更新规则。
- 人工模板尚未发布时，仍可编辑并重置为新草稿版本。
- 人工模板列表中已发布模板显示“查看详情”，其他模板显示“编辑”。
- 已发布模板的只读抽屉完整展示基础信息、变量和三端内容预览，且不显示保存或提交机翻操作。
- 多语言流程中已发布人工模板的“编辑源文案”入口禁用。

## 验收标准

1. 操作者可以查看已发布模板的完整信息和三端内容预览，但无法修改。
2. 操作者无法从人工模板列表或多语言流程修改已发布模板。
3. 直接调用更新方法也无法修改已发布的人工专用或通用模板。
4. 多语言进度和使用任务不受影响。
5. 未发布人工模板及纯事件模板保持原有编辑能力。
