# 指定 UID CSV 受众导入设计

> 日期：2026-07-14
>
> 状态：已确认
>
> 范围：消息任务前端交互原型

## 1. 背景与目标

人工发送任务已经支持选择“指定用户”并按行输入 UID，但缺少批量导入能力。本次在新建和编辑任务的目标用户步骤中增加 CSV 导入，并与手动输入 UID 合并去重。

目标：

- 操作者可以下载模板、上传 CSV、查看校验结果并确认导入。
- CSV 中有效 UID 正常导入；无效 UID 排除并要求操作者确认。
- CSV UID 与手动 UID 合并后统一去重。
- 最终目标人数、样例和任务摘要使用确认后的有效 UID 集合。
- 当前仓库只实现前端原型，不伪造真实用户服务查询。

## 2. 用户入口与页面布局

当人工任务的受众方式为“指定用户”时，在现有“用户 UID”输入区内展示 `UidAudienceImporter`：

1. 手动输入区：保持“每行一个 UID”。
2. CSV 工具栏：下载模板、选择 CSV 文件、重新上传、删除文件。
3. 上传区：支持点击选择和拖拽上传。
4. 校验结果区：展示文件名、文件总行数、有效、重复、无效和最终目标用户数。
5. 异常区：按行展示无效 UID 和原因，并提供错误明细下载。
6. 确认区：存在无效 UID 时，操作者必须点击“确认排除无效 UID”；全部有效时点击“确认导入有效 UID”。
7. 受众预览区：展示手动与 CSV 合并去重后的数量及脱敏样例。

CSV 上传区直接嵌入当前目标用户步骤，不打开抽屉、不跳转到独立受众页面。

## 3. CSV 格式

文件要求：

- 扩展名必须为 `.csv`。
- 使用 UTF-8 文本。
- 首行必须包含 `uid` 列，列名忽略首尾空格和大小写。
- 可选 `remark` 列；备注不参与受众计算和发送。
- UID 始终按字符串处理，避免长数字精度丢失。
- UID 格式为 1—64 位字母、数字、下划线或连字符，校验表达式为 `^[A-Za-z0-9_-]{1,64}$`。
- 支持 UTF-8 BOM、标准逗号分隔及双引号包裹字段；不接受无法闭合的引号。
- 前端原型默认限制文件不超过 10 MB、数据行不超过 100,000 行。

模板：

```csv
uid,remark
10000001,活动报名用户
10000002,VIP 用户
```

## 4. 数据处理规则

### 4.1 手动 UID

- 按换行符拆分。
- 去除每行首尾空格。
- 忽略空行。
- 在手动输入集合内去重。

### 4.2 CSV UID

- 解析 `uid` 列并保留原始行号。
- 空值、格式错误和超出长度限制的 UID 进入无效集合。
- 文件内重复 UID 只保留第一次出现，其他行计入重复集合。
- 再次上传 CSV 时替换上一次 CSV 数据，不叠加多个文件。

### 4.3 合并与去重

合并顺序为手动 UID 在前、CSV UID 在后。CSV 中与手动输入重复的 UID 计入重复数量，不重复进入最终集合。

```text
最终 UID = 手动有效 UID ∪ 已确认的 CSV 有效 UID
```

删除 CSV 只移除 CSV 来源 UID，保留手动输入。

### 4.4 前端原型校验边界

本次只校验文件、表头、空值和 UID 格式。正式环境中的 UID 是否存在、账户是否注销、是否被冻结、触达权限、地区限制和抑制名单由用户服务和受众服务校验。

页面明确显示：

> 当前为前端格式校验；正式发送前由用户服务再次校验存在性和可触达状态。

不得使用 UID 尾号或随机规则伪造“用户不存在”等业务结果。

## 5. 导入确认与状态

CSV 导入状态：

- `未上传`：没有 CSV 文件。
- `解析失败`：文件类型、大小、表头或 CSV 结构错误，不能确认。
- `待确认`：解析完成，尚未将有效 UID 加入最终受众。
- `已确认`：有效 UID 已加入最终受众。

行为：

- 上传成功后进入`待确认`，此时 CSV UID 不计入最终目标人数。
- 存在无效 UID 时，确认按钮为“确认排除无效 UID”。
- 全部有效时，确认按钮为“确认导入有效 UID”。
- 操作者确认后进入`已确认`并更新目标人数、样例和任务摘要。
- 手动 UID 在输入后立即参与目标人数计算。
- 修改或重新上传 CSV 后重新进入`待确认`。
- 删除 CSV 后回到`未上传`。

## 6. 组件与数据边界

新增组件：

```text
UidAudienceImporter
├── 手动 UID 输入
├── CSV 文件选择/拖拽
├── 模板与错误明细下载
├── 导入统计
├── 异常行列表
└── 导入确认
```

新增纯函数工具负责：

- CSV 文本解析。
- 表头校验。
- UID 格式校验。
- 手动与 CSV 集合合并去重。
- UID 脱敏样例生成。
- CSV 模板和错误明细内容生成。

组件通过回调向 `CreateTaskPage` 返回：

```ts
interface UidAudienceValue {
  manualUids: string[];
  csvFileName?: string;
  csvValidUids: string[];
  csvInvalidRows: Array<{ row: number; uid: string; reason: string }>;
  duplicateCount: number;
  csvConfirmed: boolean;
  finalUids: string[];
}
```

`CreateTaskPage` 使用 `finalUids.length` 作为指定 UID 受众人数，使用脱敏后的 `finalUids` 生成预览样例。

## 7. 提交和编辑规则

- 选择指定 UID 时，最终 UID 集合不能为空。
- CSV 处于`待确认`时禁止进入最终提交，并提示先确认导入结果。
- CSV 解析失败时不改变现有手动 UID 和已确认受众。
- 编辑草稿时恢复手动 UID、文件名、统计和确认状态。
- 待审核、已通过或待发送任务修改 UID 后按任务状态规则回到草稿，原审核和计划失效。
- 发送中及历史任务不能修改受众，只能查看冻结快照或复制为新任务。

当前前端原型将最终 UID、文件名和统计信息保存在任务草稿状态中；生产系统应保存不可变 `audience_snapshot_id` 和源文件哈希，不在浏览器长期保存原始 CSV。

## 8. 异常与安全

- 缺少 `uid` 表头：拒绝解析并提示下载模板。
- 非 CSV 文件：拒绝选择。
- 空文件或只有表头：提示没有可导入 UID。
- 超过大小或行数限制：拒绝导入。
- CSV 解析异常：保留手动 UID，展示错误，不产生部分导入。
- 无效 UID：排除并展示行号、脱敏值和原因。
- 重复 UID：自动去重并计数，不作为错误阻断。
- 页面和错误明细默认对 UID 脱敏；原始 UID 仅在操作者输入区和本地解析过程中使用。
- 不接受手机号、邮箱、姓名等替代 UID 字段。

## 9. 测试与验收

自动化测试至少覆盖：

1. 选择“指定用户”后展示手动输入和 CSV 导入。
2. 正确解析包含 `uid` 表头的 CSV。
3. 缺少 `uid` 表头时拒绝导入。
4. 无效 UID 被排除并展示原因。
5. 文件内重复 UID 自动去重。
6. CSV UID 与手动 UID 合并去重。
7. 未确认 CSV 时不计入目标人数并阻止最终提交。
8. 确认后更新最终人数和脱敏样例。
9. 删除 CSV 后保留手动 UID。
10. 重新上传替换上一份 CSV。
11. 模板和错误明细可以下载。
12. 其他受众类型和系统事件任务不展示 CSV 导入。

页面验收：

- 导入结果在目标用户步骤内完整可见，不依赖提示弹窗。
- 有效、重复、无效和最终人数之间关系明确。
- 所有按钮可操作，空态、错误态、待确认和已确认状态可辨识。
- 最终任务摘要与受众预览使用相同的合并去重 UID 集合。

## 10. 非本次范围

- 真实用户存在性和可触达状态查询。
- 服务端文件存储、病毒扫描和对象存储。
- 可复用 UID 名单管理。
- 超过 100,000 UID 的异步受众计算。
- 生产受众快照接口和发送服务接入。
