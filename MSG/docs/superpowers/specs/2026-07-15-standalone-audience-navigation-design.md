# “用户与受众”独立导航设计

## 目标

将“用户与受众”从“人工消息”二级菜单中移出，调整为与“工作台”“系统配置”同级的一级菜单入口，让受众资产不再被误解为仅服务人工消息。

## 信息架构

调整后的主导航顺序：

1. 工作台
2. 用户与受众
3. 人工消息
   - 人工消息任务
   - 人工消息模板
4. 事件自动化
   - 事件通知规则
   - 事件消息模板
   - 事件目录
   - 触发记录
5. 运营与治理
   - 多语言审核
   - 审核中心
   - 发送记录
   - 数据分析
6. 系统配置

## 路由与页面

- 保留现有路由 `/segments`，不修改用户与受众页面内容。
- “用户与受众”使用现有用户群组图标。
- 当前路由为 `/segments` 或其子路径时，一级菜单“用户与受众”处于选中状态。
- 面包屑显示“消息中心 / 用户与受众”，不再包含“人工消息”。
- 侧栏折叠时仍显示“用户与受众”图标，并支持点击进入原页面。

## 兼容范围

- 不修改人工消息任务中的受众选择能力。
- 不修改用户分群数据、创建流程或页面交互。
- 不修改多语言进度、翻译结果或多语言审核流程。
- 不新增路由，不迁移历史数据。

## 验收标准

1. “用户与受众”作为独立一级菜单显示在“工作台”之后。
2. “人工消息”分组只包含“人工消息任务”和“人工消息模板”。
3. 访问 `/segments` 时菜单选中和面包屑正确。
4. 原有 `/segments` 页面可正常打开。
5. 导航单元测试、完整测试和生产构建通过。
