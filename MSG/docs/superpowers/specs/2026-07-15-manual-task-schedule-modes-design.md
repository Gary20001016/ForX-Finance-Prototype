# 人工消息任务发送模式收敛设计

## 目标

人工消息任务仅支持“立即发送”和“指定时间发送”，计划发送时间只在指定时间模式下展示并必填。

## 交互规则

- 新建任务默认选择“立即发送”。
- 发送模式下拉仅包含“立即发送”和“指定时间发送”。
- 立即发送时隐藏计划发送时间，并清除表单中已有的计划时间。
- 指定时间发送时显示计划发送时间，并要求操作者选择后才能进入下一步或提交。
- 删除“用户本地时间”“保留原计划”和“本地发送时间”字段。
- 编辑历史任务时，原排期为“立即”或“立即发送”映射为立即发送；其他历史排期映射为指定时间发送，并要求重新确认时间。
- 历史任务记录保持原展示，不批量修改历史数据。

## 数据规则

- 立即发送统一保存为`立即`，与任务审核通过后的自动启动判断一致。
- 指定时间保存日期时间字符串。
- 未选择指定时间时显示“指定时间待填写”，但禁止完成提交。
- 事件自动化继续使用“事件到达时”，不使用人工排期字段。

## 非范围

- 不支持用户本地时间、周期发送或重复任务。
- 不修改已经存在的任务记录。
- 不修改事件通知规则的触发时间策略。

## 验收标准

1. 人工任务发送模式只有两个选项。
2. 默认立即发送时不展示计划发送时间。
3. 选择指定时间后展示必填时间字段。
4. 切回立即发送后计划时间被清除。
5. 立即发送任务保存值为`立即`。
