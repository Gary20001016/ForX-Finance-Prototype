# 嵌套组合分群设计

## 目标

允许操作者把已有组合分群与其他动态分群、静态分群或组合分群再次组合，保存为新的组合分群。

## 交互设计

- 创建组合分群时，来源选择器展示全部已有分群，不再排除“组合分群”。
- 每个选项展示分群名称、类型和当前人数，帮助操作者识别来源。
- 已有组合分群在表达式中作为一个整体，并保留括号。例如：`（沉默用户 ∩ ETH 活跃用户）∪ 高风险名单`。
- 预估人数继续基于各直接来源分群的当前人数进行前端原型计算。
- 至少选择两个来源分群后才能保存。

## 依赖与循环处理

- 新建分群没有自身引用风险，可选择任意已有分群。
- 编辑已有组合分群时，不能选择自身。
- 如果某候选分群直接或间接引用当前分群，该候选项不可选，避免形成循环依赖。
- 当前分群已有但已变成非法的来源应从选择结果中移除，不能保存循环结构。

## 数据结构

沿用现有 `sourceSegmentIds` 和 `setOperation` 字段。嵌套关系通过来源 ID 构成依赖图，不新增数据字段。

## 验证范围

- 新建组合分群时可以选择已有组合分群。
- 嵌套表达式正确保留括号和内部运算关系。
- 编辑时排除自身与会形成循环依赖的候选分群。
- 现有普通并集、交集与人数预估行为保持不变。
