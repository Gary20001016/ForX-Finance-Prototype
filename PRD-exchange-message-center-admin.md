# 交易所统一消息中心后台 PRD

> 文档版本：V1.0  
> 文档日期：2026-07-12  
> 产品范围：后台管理端  
> 已确认方向：站内信、App Push、邮件、短信；运营群发、系统事件、营销自动化；多语言和分地区合规；风险分级审核

---

## 1. 摘要

建设一个统一的交易所消息中心后台，让运营、产品、风控、合规和技术团队在同一平台管理消息模板、目标用户、自动化流程、审批、发送和数据分析。所有渠道共用统一任务与审计模型，同时保留各渠道的内容、供应商和送达特性。

系统必须确保资金、安全和合规消息可靠送达，营销消息遵守用户授权、退订、频控、安静时段及地区规则，并能够追溯“谁在什么时间向哪些用户发送了什么内容”。

## 3. 背景与问题

交易所的消息通常分散在业务服务、运营工具和多个渠道供应商中，容易出现以下问题：

- 同一活动需要分别配置站内信、Push、邮件和短信，内容及用户范围不一致。
- 充值、提现、登录安全等通知缺少统一送达追踪和失败补偿。
- 无法统一控制用户频次、安静时段、退订和国家/地区限制。
- 模板修改、审批、发布与实际发送内容无法形成完整证据链。
- 运营只看到发送量，难以分析打开、点击和最终交易转化。
- 多业务同时触达用户，容易重复轰炸或让营销通知挤占安全通知。

本项目通过“统一模板—统一任务—统一用户实例—多渠道发送实例”解决上述问题。

## 4. 目标与成功指标

### 4.1 产品目标

1. 提供统一、可审核、可追踪的多渠道消息生产与发送能力。
2. 保证强事务消息高可靠送达，支持渠道降级和失败补偿。
3. 支持多语言、多地区、用户偏好、授权、退订和频控。
4. 支持运营群发、业务事件触发和营销自动化三类场景。
5. 建立从发送、送达、打开、点击到业务转化的分析闭环。

### 4.2 建议关键结果

| 指标 | 首版目标 | 统计说明 |
|---|---:|---|
| 强事务消息任务创建成功率 | ≥99.99% | 已接收事件中成功创建用户消息实例的比例 |
| 站内信写入成功率 | ≥99.99% | 排除无效用户 |
| Push/邮件供应商受理率 | ≥99.5% | 排除用户无有效地址或 Token |
| 短信供应商受理率 | ≥99% | 按国家和供应商统计 |
| 可归因发送覆盖率 | 100% | 每条发送可关联任务、模板版本和用户 |
| 高风险任务审批覆盖率 | 100% | 不允许绕过双审 |
| 营销退订执行正确率 | 100% | 退订用户不得再收到对应营销类别 |
| 后台关键操作审计覆盖率 | 100% | 创建、修改、审批、发布、暂停、导出、配置变更 |

| 性质 | 示例 | 用户可退订 | 安静时段 | 审核建议 |
|---|---|---|---|---|
| 强事务 | 登录异常、提现、密码/2FA 变更 | 否 | 可突破 | 模板双审；事件自动发送 |

## 6. 价值主张

- 对用户：重要安全和资金消息及时、准确；营销消息可控制，语言和时间更合适。
- 对运营：一次创建即可覆盖多个渠道和语言，能看到实际触达和转化。
- 对风控合规：发送前有策略拦截，发送后有不可抵赖的审计证据。
- 对技术团队：业务只接事件与模板契约，无需分别维护四套供应商逻辑。
- 对管理层：统一查看成本、触达质量、异常和业务贡献。

## 7. 方案设计

### 7.1 总体架构与数据流

```text
运营群发 ─┐
业务事件 ─┼→ 模板与变量渲染 → 受众计算 → 合规/偏好/频控 → 用户消息实例
自动化流程 ┘                                                   ↓
                                站内信 / Push / 邮件 / 短信发送实例
                                               ↓
                           受理 / 送达 / 打开 / 点击 / 转化 / 退订
```

核心原则：

1. 模板版本、任务版本、审批版本相互绑定；审核后任何内容变更都必须重新审核。
2. 用户消息实例保存最终渲染快照，历史模板变化不能影响审计。
3. 渠道发送实例独立记录状态、供应商、成本、失败原因和重试次数。
4. 事务与营销使用同一基础设施，但优先级、频控、授权和降级规则不同。
5. 用户明细默认脱敏；导出与查看完整 PII 需要独立权限。

### 8.2.8 提交审核前检查

系统必须展示并阻止以下问题：

- 缺失语言或渠道内容、变量未映射、变量类型不匹配。
- 深链不在白名单、附件不安全、内容命中禁止词。
- 目标地区无合法渠道或内容未取得地区批准。
- 预计受众为零、受众计算过期、上传名单含越权用户。
- 计划时间早于审批 SLA、周期任务无结束条件。
- 营销消息未配置退订入口、短信缺签名、邮件缺发件人。
- 预算不足、渠道不可用、成本超出任务阈值。

提交确认页展示：最终内容预览、差异、预计人数、过滤人数、预计成本、时间、频控、风险等级和完整审批链。

### 8.2.9 任务状态机

```text
草稿 → 校验中 → 待一级审核 → 待二级/合规/风控审核 → 审核通过
                                                     ↓
            已取消 ← 待发送 → 发送中 ↔ 已暂停 → 已完成/部分完成/失败
                         ↓
                       已过期

任一审核节点 → 已驳回 → 修改后重新提交
审核通过且未发送 → 可撤回为草稿；任何修改使原审批失效
```

状态定义：

| 状态 | 含义 | 允许操作 |
|---|---|---|
| 草稿 | 尚未提交 | 编辑、删除、复制、测试、提交 |
| 校验中 | 正在计算受众和规则 | 查看、取消校验 |
| 审核中 | 等待审批 | 查看、撤回；审核人可通过/驳回 |
| 审核通过 | 全部审批完成 | 查看、撤回、取消 |
| 待发送 | 等待调度时间 | 查看、取消、授权修改时间 |
| 发送中 | 正在生成或发送 | 查看、暂停、紧急停止 |
| 已暂停 | 人工或熔断暂停 | 查看、恢复、取消 |
| 已完成 | 所有有效实例处理结束 | 查看、复制、归档 |
| 部分完成 | 存在永久失败/过期 | 查看、重试可重试失败 |
| 失败 | 任务级故障 | 查看原因、重试、复制 |
| 已取消 | 人工终止 | 查看、复制 |
| 已过期 | 未在有效期内发送 | 查看、复制 |

## 8.7 审核中心

### 8.7.1 审批规则

| 风险 | 示例 | 审批链 |
|---|---|---|
| 低 | 小范围普通运营 | 一级审核 |
| 中 | 较大范围、受限地区、周期任务 | 一级审核 + 业务负责人 |
| 高 | 全量群发、资金/风控、较高成本 | 一级 + 二级 + 风控或合规 |
| 关键 | 紧急安全、全站公告、突破安静时段 | 双审 + 指定授权人；紧急通道 |

风险自动升级条件包括：全量用户、预计人数超阈值、短信成本超阈值、强事务、敏感变量、受监管地区、突破安静时段、外部链接、临时名单、首次使用的新模板或新供应商。

### 8.7.2 审核列表字段

`审批单ID`、`对象类型`、`对象名称/ID`、`版本`、`风险等级`、`消息性质`、`预计人数`、`预计成本`、`计划时间`、`当前节点`、`剩余SLA`、`提交人`、`提交时间`、`审批状态`、`操作`。

### 8.7.3 审核详情必须展示

- 基础信息、发送原因、关联工单、风险标签。
- 各语言和渠道的最终内容预览及上一版本差异。
- 变量定义、映射、示例渲染和敏感信息提示。
- 受众条件、排除条件、人数漏斗和随机样本。
- 地区、用户偏好、退订、频控、安静时段的执行结果。
- 渠道顺序、降级策略、预计成本、预算余额和熔断阈值。
- 计划时间、有效期、测试发送结果。
- 已有审批意见、操作日志和申请人历史驳回记录。

### 8.7.4 审批操作字段

| 字段 | 通过 | 驳回 | 转交/加签 |
|---|---:|---:|---:|
| 审批结论 | 是 | 是 | 是 |
| 审批意见 | 高风险必填 | 必填 | 必填 |
| 驳回原因分类 | 否 | 必填 | 否 |
| 风险确认勾选 | 高/关键必填 | 否 | 否 |
| 指定处理人 | 否 | 否 | 必填 |
| 附件/证据 | 否 | 否 | 否 |
| 二次验证 | 关键任务必填 | 否 | 权限变化时必填 |

创建人不能审核自己的对象；同一人不能完成两个强制独立节点。审批通过后系统计算内容和配置哈希，发送前再次校验，不一致则自动使审批失效。

### 8.7.5 紧急发送

紧急通道不取消审批，只缩短路径。申请字段：`紧急原因`、`影响说明`、`不立即发送的风险`、`目标范围`、`渠道`、`突破的规则`、`授权人`、`关联事故单`、`预计有效期`。紧急任务必须使用已发布模板，禁止临时富文本和未备案外链；发送后自动生成复盘工单。

### 8.8.3 失败重试规则

- 临时失败按指数退避并加入随机抖动；不得超过任务有效期和最大次数。
- 永久失败（无效号码、退订、硬退信、无效 Token）不自动重试，并更新渠道地址状态。
- 人工批量重试必须选择失败码范围、目标渠道和有效期，显示预计人数与成本；高风险重试需审批。
- 供应商回调必须验签、幂等处理，并保留原始回执摘要。

导出申请包含：`导出原因`、`字段范围`、`时间范围`、`数据量预估`、`是否含 PII`、`审批人`、`文件有效期`。文件加密、有水印、限制下载次数；默认只允许脱敏 UID、状态和错误码。

## 10. 关键业务规则

1. **去重与幂等**：事件通过业务幂等键去重；群发按 `task_id + user_id` 去重；渠道重试不得新建用户消息实例。
2. **语言回退**：用户语言精确匹配 → 同语言通用地区版本 → 任务默认语言 → 模板默认语言；强事务无可用回退则告警并进入死信，营销可直接跳过。
3. **用户地区**：优先使用合规确认的居住/KYC 地区，其次注册地区；不得仅依赖 IP 临时位置决定法律规则。
4. **用户时区**：用户设置 → 最近可靠设备时区 → 地区默认时区 → 任务时区。
5. **营销授权**：发送前实时校验授权和退订；即使审批时允许，发送前已退订也必须拦截。
6. **优先级隔离**：强事务与营销使用独立队列、配额和熔断，营销流量不得阻塞安全消息。
7. **渠道降级**：只有任务明确配置并满足地区规则时才降级；不得把退订邮件降级为短信营销。
8. **内容冻结**：发送实例必须关联已审批内容哈希；哈希不一致时停止发送并告警。
9. **删除策略**：任务、模板、审批、发送和审计记录不允许物理删除；使用停用或归档。
10. **时间一致性**：数据库统一存 UTC，后台按管理员/任务/用户时区显示并明确标注。

## 11. 异常处理与告警

| 异常 | 系统动作 | 告警对象 |
|---|---|---|
| 事件 Schema 校验失败 | 拒绝并返回明确错误；按阈值告警 | 调用方、技术值班 |
| 重复事件 | 返回幂等成功，不重复发送 | 仅计数，异常激增时告警 |
| 受众计算失败 | 阻止提交或发送 | 任务创建人、数据平台 |
| 模板渲染失败 | 单用户进入死信；超过阈值熔断任务 | 创建人、技术值班 |
| 单供应商失败率升高 | 熔断并切备用供应商 | 渠道管理员 |
| 所有供应商不可用 | 暂停对应渠道；强事务尝试合法降级 | 技术、风控 |
| 队列积压 | 扩容/限流营销；保护强事务 | 技术值班 |
| 退订/投诉率超阈值 | 自动暂停营销任务 | 运营、合规 |
| 审批后配置哈希变化 | 阻止发送并使审批失效 | 创建人、审核人 |
| 成本或余额异常 | 限制任务并告警 | 运营、财务、渠道管理员 |

告警字段：级别、类型、对象、指标值、阈值、影响范围、开始时间、负责人、处理状态、确认时间、恢复时间和事故单。

## 12. 非功能需求

### 12.1 性能与容量

- 支持水平扩展；强事务与营销队列隔离。
- 容量指标由技术评审根据交易所用户量确认；首版压测至少覆盖预估峰值的 2 倍。
- 任务列表、发送记录常规查询 P95 不超过 2 秒；大范围导出采用异步任务。
- 百万级群发不得同步生成全部数据库记录后才开始发送，应采用分片和流式处理。

### 12.2 可用性与容灾

- 消息接收、调度、发送、回执分别具备幂等与重放能力。
- 跨可用区部署；关键队列和数据库有备份、监控和恢复演练。
- 供应商故障不得导致事件丢失；死信可查询、可授权重放。

### 12.3 安全

- 管理端强制 SSO、MFA、最小权限和敏感操作二次验证。
- API 使用服务身份、签名、时间戳、防重放和速率限制。
- 手机号、邮箱、事件敏感载荷加密；日志不得保存明文凭证和完整 PII。
- 供应商密钥存放于密钥管理系统，支持轮换和到期告警。
- 导出文件加密、限时、限次下载并记录水印。

### 12.4 数据保留

不同地区和数据类型使用版本化保留策略。审批证据、发送状态和审计日志按法规及公司政策保留；渲染内容中的敏感变量应尽量引用或加密，并在保留期结束后匿名化，而非影响汇总报表。

## 13. 验收标准与测试场景

### 13.1 核心功能验收

1. 可创建包含四个渠道、至少三种语言的定时群发任务，并完成测试发送、受众预估和审批。
2. 审核后修改任一内容、受众、时间、渠道或合规配置，原审批自动失效。
3. 同一提现事件重复调用时只生成一个用户消息实例。
4. 营销退订用户在任务审批后、实际发送前退订，系统仍能实时拦截。
5. 用户处于安静时段时，营销消息延迟；授权的安全消息按策略立即发送。
6. Push 永久失败后可按合法策略降级到短信，且每一步可追踪。
7. 某短信供应商达到熔断阈值后自动切换备用供应商，不重复计费或重复发送。
8. 全量用户任务无法由创建人自审，且必须完成双审。
9. 可从任务追踪到模板版本、审批链、用户实例、供应商回执和转化事件。
10. 无 PII 权限人员看到脱敏数据，导出 PII 必须审批并产生审计日志。

### 13.2 边界测试

- 缺失语言、变量为空、超长短信、无效深链、过期模板、受众为零。
- 跨夏令时发送、用户时区缺失、周期任务跨月和节假日。
- 一名用户同时属于多个分群、同时进入多个流程、同一渠道有多个 Token。
- 任务发送中暂停、供应商迟到回执、回执重复或乱序。
- 退订与发送并发、模板停用与事件并发、审批撤回与调度并发。
- 大规模名单上传、部分无效 UID、重复 UID、恶意 CSV 内容。

### 13.3 性能与安全测试

- 峰值事件吞吐、百万级受众计算、队列积压恢复和批量回执。
- 供应商超时、限流、部分故障、全故障和恢复后的重试风暴。
- 越权查看、越权导出、自审、审批链绕过、凭证泄露、回调伪造。
- 富文本 XSS、邮件 HTML 注入、深链参数注入、CSV 公式注入。

## 14. 发布计划

### 14.1 第一阶段：统一事务通知基础能力

- 模板、多语言、变量、事件接入。
- 站内信、Push、邮件、短信四渠道。
- 单/双审、发送记录、失败重试、基础渠道管理。
- 账户安全、资金和系统公告优先迁移。

### 14.2 第二阶段：运营群发与全球合规

- 用户分群、定时/分批群发、测试发送。
- 地区、授权退订、安静时段、频控和黑名单。
- 数据分析、成本统计和渠道路由。

### 14.3 第三阶段：营销自动化与高级分析

- 自动化流程画布、A/B 分流、转化归因。
- 高级熔断、供应商智能路由和异常检测。
- 更多渠道扩展，如 Telegram、Webhook 等。

每阶段都先迁移少量低风险业务，经过灰度、并行核对和回滚演练后再扩大范围。旧发送系统下线前必须完成模板、事件、回执和用户偏好的数据对账。

## 15. 假设、风险与待项目评审项

### 15.1 已采用的产品假设

- 交易所使用统一 UID，且可读取用户语言、地区、时区、渠道地址和通知偏好。
- 业务系统能够通过稳定事件编码和幂等键接入。
- 公司已有 SSO、RBAC、密钥管理、用户画像/标签和埋点平台，消息中心通过接口集成。
- 不在消息模板或事件载荷中传输私钥、验证码种子等高危数据。

### 15.2 主要风险

| 风险 | 缓解措施 |
|---|---|
| 全球法规差异且持续变化 | 合规策略版本化、定期复核、按地区灰度 |
| 多语言内容缺失或误译 | 默认语言回退、翻译角色、预览与差异审核 |
| 供应商状态不一致 | 统一状态映射、原始回执留存、渠道级报表 |
| 大规模群发影响事务通知 | 队列和配额隔离、优先级、限流和熔断 |
| 用户画像数据过期 | 展示数据时点；高风险任务冻结名单并发送前复核 |
| 自动化流程造成重复触达 | 固定分桶、重复进入规则、全局频控、幂等 |
| 指标归因夸大效果 | 明确窗口和模型，提供对照组及口径版本 |

### 15.3 立项时需补充但不阻塞本方案的参数

以下不是产品结构上的空缺，需要结合真实业务量和公司制度在技术/合规评审时填入配置：峰值 TPS、总用户量、单次最大群发量、审批 SLA、短信预算阈值、各地区保留期、具体供应商、目标 RTO/RPO 和各渠道成本。

## 16. 明确不在本后台首版范围

- 客服与用户实时双向聊天。
- 社交媒体私信和群机器人运营。
- 广告投放平台和完整 CDP 用户画像建设。
- 由消息中心直接执行交易、提现或账户权限变更。
- AI 自动生成并绕过人工审核后直接发布内容。
- 对交易所全部业务埋点和转化数据仓库进行重建。

## 附录 A：后台操作硬规则

1. 创建人不得审核自己的任务、模板或自动化流程。
2. 高风险和关键消息的两个审批节点必须由不同人员完成。
3. 全量用户、资金安全、风控合规、突破安静时段和超预算任务自动升级审核。
4. 审批通过后内容、变量、受众、渠道、时间和策略全部锁定；任何修改重新审核。
5. 测试发送必须清晰标注测试环境，禁止测试消息误发真实用户。
6. 强事务消息不受营销退订影响，但仍受法律禁止、地址无效和安全策略约束。
7. 查看完整手机号、邮箱、原始事件载荷和供应商回执需要单独权限。
8. 任务停止、批量重试、PII 导出、供应商切换和紧急发送必须记录原因。
9. 所有后台时间必须显示时区；数据库统一存 UTC。
10. 所有列表导出、批量操作和配置发布均采用异步任务并可追踪结果。

## 附录 B：建议统一错误码分类

| 分类 | 示例 | 是否重试 |
|---|---|---|
| VALIDATION | 缺变量、格式错误、内容超长 | 否，修复数据 |
| COMPLIANCE | 未授权、退订、地区禁止、频控 | 否或延迟 |
| ADDRESS | 无 Token、无效邮箱/号码、硬退信 | 否，并抑制地址 |

| PROVIDER_PERM | 模板未备案、签名错误、账号停用 | 否，告警管理员 |
| EXPIRED | 超过任务或渠道 TTL | 否 |
| INTERNAL | 渲染、队列、存储异常 | 是；超过阈值熔断 |
