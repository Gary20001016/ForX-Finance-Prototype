# 人工消息受控模板变量设计

## 目标

为人工消息提供安全、统一的模板变量能力。人工模板和临时消息的操作者只能从受控变量库搜索、复制或插入变量，不能手工声明变量；变量在机器翻译、人工校对和多语言发布过程中保持原样。

## 范围

- 在“人工消息”导航分组下新增“模板变量”页面。
- 人工消息模板和人工临时消息共用同一套变量选择、插入和校验规则。
- 变量仅能插入站内信正文与 App Push 正文。
- 支持有权限操作者单条维护及 CSV 导入。
- 系统事件模板继续使用事件字段和变量映射，不接入人工消息变量库。

## 信息架构

“人工消息”分组包含：

1. 人工消息任务
2. 人工消息模板
3. 模板变量

“模板变量”是人工消息的受控公共资源，不作为系统配置或事件自动化的一部分。

## 数据模型

受控变量包含以下字段：

| 字段 | 含义 | 规则 |
| --- | --- | --- |
| `id` | 内部标识 | 系统生成 |
| `name` | 变量名 | `snake_case`；创建后不可修改 |
| `description` | 变量说明 | 面向操作者解释变量含义 |
| `status` | 状态 | 启用、停用 |
| `updatedAt` | 更新时间 | 系统记录 |
| `updatedBy` | 更新人 | 系统记录 |

正文占位符格式统一为 `{{ variable_name }}`。变量实际值与解析逻辑由后台维护，前端不展示或编辑变量值及数据来源。

## 模板变量管理页

### 查询与展示

- 支持按变量名或说明搜索。
- 支持按启用、停用状态筛选。
- 列表展示变量名、变量说明、状态、更新时间、更新人和操作。
- 普通人工消息操作者可以查看并复制完整占位符。

### 权限

- 查看、搜索、复制与正文插入向人工消息操作者开放。
- 新增、CSV 导入、修改说明、启用和停用仅向拥有变量维护权限的操作者开放。
- 无维护权限时不展示管理按钮。
- 变量名创建后不可修改；已使用变量不可删除，只能停用。

### 单条维护

- 新增时填写变量名和变量说明，状态默认为启用。
- 编辑时只能修改变量说明和状态。
- 停用前提示该变量将不能继续用于新消息提交，但不破坏历史消息快照。

### CSV 导入

CSV 仅接受：

```csv
variable_name,description
user_nickname,用户昵称
vip_level,用户当前 VIP 等级
```

- 导入前解析并展示新增数量、更新数量和错误数量。
- 新变量名创建变量，状态默认为启用。
- 重复变量名仅更新说明，不创建重复记录。
- 变量名格式错误、说明为空或同一文件内重复的记录进入错误明细。
- 操作者确认预览后才应用导入结果。

## 正文插入体验

### 支持位置

- 站内信 Markdown 正文。
- App Push 正文。

标题、摘要、风险提示、按钮文案和跳转链接不提供变量入口。

### 变量选择器

- 站内信 Markdown 工具栏增加“插入变量”。
- App Push 正文旁增加“插入变量”。
- 点击后打开可复用的变量选择器。
- 支持按变量名或说明搜索，仅展示启用变量。
- 每条变量展示完整占位符和说明，并提供“插入正文”和“复制变量”。
- “插入正文”写入当前光标位置；没有有效光标时追加到正文末尾。
- “复制变量”复制完整 `{{ variable_name }}`。

人工模板编辑器原有的“模板变量”手工输入框删除。人工临时消息使用相同选择器和插入规则。

## 校验与多语言规则

### 提交校验

提交草稿以外的审核动作前，系统扫描两个正文中的 `{{ ... }}`：

- 变量存在且启用：通过。
- 变量不存在：禁止提交，并列出无效变量名。
- 变量已停用：禁止提交，并列出停用变量名。
- 格式不完整：禁止提交，并提示修复占位符。

保存草稿允许暂存无效内容，但需展示问题提示。

### 多语言保护

- 向外部机器翻译任务提交内容时保护变量占位符。
- 翻译结果返回后校验每个变量的名称与出现次数。
- 变量占位符在源语言、机器翻译、人工修订和最终预览中保持完全一致。
- 变量名称或数量发生变化时，该语言不能通过审核。

## 前端组件边界

- `TemplateVariablePage`：变量查询、筛选、单条维护和 CSV 导入。
- `VariablePicker`：变量搜索、复制与插入，供人工模板和临时消息复用。
- 变量校验模块：提取正文占位符并对照受控变量库校验。
- 变量 CSV 模块：解析、校验、统计和生成错误明细。
- 原型数据存储：保存变量列表及新增、编辑、启停和批量导入操作。

各模块使用统一变量数据模型，编辑器不自行维护变量列表或重复实现校验逻辑。

## 错误处理

- CSV 文件缺少表头、编码异常或无法解析时，不覆盖当前变量数据。
- CSV 存在部分错误时允许预览，但确认导入只应用有效记录并明确显示错误明细。
- 复制失败时显示错误提示，不改变正文。
- 插入变量失败时保留原正文，并提示重新操作。
- 提交校验失败时停留在内容步骤，并显示具体变量及所在渠道。

## 验收标准

1. “人工消息”菜单下可进入“模板变量”页面。
2. 普通操作者能搜索、查看和复制变量，但看不到维护操作。
3. 有权限操作者能新增、编辑说明、启停变量并完成 CSV 预览导入。
4. 人工模板和临时消息都能在站内信正文及 Push 正文插入变量。
5. 其他内容字段没有变量插入入口。
6. 人工模板中不再出现手工声明模板变量的输入框。
7. 无效、停用或格式错误的变量会阻止提交审核。
8. 所有目标语言的变量名称与数量保持一致，并可在预览中看到原始占位符。
9. 事件模板的事件变量与映射流程不受影响。

## 非目标

- 不在前端维护变量实际值或解析逻辑。
- 不允许操作者自行输入未登记变量。
- 不允许通过 CSV 修改已经存在的变量名。
- 不支持在标题、摘要或按钮文案中插入变量。
- 不改造系统事件变量管理。
