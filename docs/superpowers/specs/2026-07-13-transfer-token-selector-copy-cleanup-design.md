# App 划转代币选择与文案清理设计

## 目标

在现有 ForX App 划转 HTML 原型上完成一次聚焦调整：删除无业务价值的装饰英文和划转历史入口，并将固定 USDC 改为可用的 USDC/USDT 代币下拉选择。继续只保留资金账户与合约账户。

## 界面调整

- 删除主原型中的 `Capital route`，仅保留“资金路线”。
- 删除原型展示页中的 `INTERACTIVE PROTOTYPE`、`SCREEN BREAKDOWN`、`ERROR STATES` 等装饰英文；需要保留的分区标题使用中文。
- 删除 App 顶栏右侧的划转历史按钮及图标，不保留空白占位；标题保持视觉居中。
- 资金路线标题右侧放置代币选择控件，默认显示 USDC。控件包含代币圆形标识、代币代码和下拉箭头。
- 点击代币控件打开轻量下拉菜单，提供 USDC、USDT 两项。当前项显示选中状态。
- 下拉菜单支持鼠标、触摸、键盘、Escape 关闭和点击外部关闭。

## 状态与数据

- 代币状态为 `USDC` 或 `USDT`。
- 两种代币使用独立的账户余额，避免切换代币后沿用另一代币的划转结果：
  - USDC：资金账户 20，合约账户 0。
  - USDT：资金账户 50，合约账户 0。
- 最低划转金额均为 10，手续费均为 0。
- 切换代币时清空金额输入、校验提示与待处理操作，并重新渲染当前代币的账户余额。
- 切换账户方向时只交换当前代币的转出/转入账户，不影响另一代币余额。

## 全流程同步

选中代币必须同步到以下位置：

- 账户余额与可用余额
- 金额输入框右侧单位
- 最低划转提示与错误文案
- 预计到账金额
- 核对弹层的金额、余额与手续费
- 处理中业务状态
- 成功回执和成功后的两账户余额
- 失败后重试操作
- 静态页面拆解与错误状态画廊

划转成功只修改当前代币对应的余额；切换到另一代币后不应看到余额被串改。

## 交互与异常处理

- 代币菜单打开时设置正确的 `aria-expanded`，菜单使用可理解的语义与焦点顺序。
- 当前处于核对、处理或失败状态时不提供代币切换；只有返回编辑页后可切换。
- 服务不可用时仍允许查看和切换代币，但检查划转按钮保持禁用。
- 重置原型恢复 USDC、资金账户转出、USDC/USDT 的初始独立余额和空金额。

## 测试与验收

- HTML 中不存在被指定删除的装饰英文和划转历史控件。
- 下拉菜单包含且仅包含 USDC、USDT。
- 代币切换会更新单位、余额、提示并清空金额。
- USDT 可完成一次完整划转，核对、回执和余额均显示 USDT。
- USDT 划转后切回 USDC，USDC 余额保持不变。
- Reset 同时恢复默认代币与两种代币余额。
- 原有金额校验、正反向划转、失败重试、服务不可用、焦点陷阱、320px 响应式与无外部请求测试继续通过。

## 范围外

- 不接入真实资产接口。
- 不增加其他账户类型。
- 不增加划转历史页面或入口。
- 不增加除 USDC、USDT 之外的代币。
