# 单语言直接编写与语言审核设计

## 目标

允许操作者直接使用目标语言创建单语言消息模板或临时消息。例如只需要日文内容时，可将 `ja-JP` 设为默认语言并直接编写日文，不发起“日文到日文”的机器翻译任务。

同时保留小语种审核门禁：若默认语言在“多语言审核配置”中被标记为需要专项审核，则单语言原文仍须进入对应语言审核组，审核通过后才能进入业务审核。

## 方案选择

采用显式的“单语言直接编写”生产模式，不使用以下替代方案：

- 不采用“目标语言为空即直接通过”，因为这会绕过日语等专项审核语言的人工审核。
- 不创建同语言伪机翻任务，因为外部任务编号、机翻次数和机翻结果等字段会误导操作者，也会污染审计记录。

## 核心规则

1. 默认语言可以选择系统支持的任意语言，包括 `ja-JP`。
2. 目标语言允许为空。为空表示当前消息模板或临时消息是单语言内容。
3. 目标语言不得包含默认语言；修改默认语言时，系统自动从目标语言中移除该语言。
4. 有目标语言时，沿用现有外部异步机器翻译流程。
5. 无目标语言且默认语言需要专项审核时，创建“原文审核”批次，不调用外部翻译接口。
6. 无目标语言且默认语言不需要专项审核时，语言准备直接通过；临时消息可继续配置人工任务，消息模板可进入业务审核。
7. 后续为单语言内容增加目标语言时，以当前默认语言原文为源内容发起外部机翻。

## 数据模型

### 语言生产模式

翻译批次和语言条目增加生产模式：

```ts
type LanguageProductionMode =
  | "machine_translation"
  | "direct_source_review";
```

`machine_translation` 表示现有的外部机翻流程；`direct_source_review` 表示操作者直接编写默认语言内容，并将原文送审。

### 原文审核条目

原文审核条目复用现有语言审核能力，但具备以下差异：

- 审核语言等于默认语言。
- 不生成 `externalTaskId`。
- `attemptNo` 为 `0`。
- 不保存或展示机器翻译结果。
- 人工修订稿初始值来自默认语言原文。
- 审核组、审核人数、提交人回避和审核 SLA 继续读取该语言的审核策略。

为兼容现有三状态模型，底层仍使用统一审核门槛；界面根据生产模式显示不同文案：

- 机器翻译：`无结果 → 翻译返回待审核 → 已通过`
- 单语言直写：`原文待审核 → 已通过`

“原文待审核”是界面语义，不新增额外的机器翻译过渡状态。

## 页面与组件调整

### 新增消息模板

- 默认语言下拉框支持所有已启用语言。
- 目标语言选择器允许留空，并显示“可留空，表示单语言模板”。
- 当目标语言为空时，显示“单语言模板，无需机器翻译”。
- 默认语言需要专项审核时，主操作显示“提交语言审核”。
- 默认语言不需要专项审核时，主操作显示“保存并进入业务审核”。
- 有目标语言时，主操作继续显示“提交外部机翻”。
- 内容编辑模块仍严格跟随正式渠道选择，仅校验和提交已选择渠道的内容。

### 新建人工消息任务中的临时消息

- 操作者选择“临时消息”后，可以选择默认语言并直接编写该语言内容。
- 目标语言允许留空；留空时将当前任务内容标记为单语言直接编写。
- 默认语言需要专项审核时，主操作显示“提交语言审核”，并为当前人工消息任务内容创建原文审核批次。
- 默认语言不需要专项审核时，语言准备直接完成，操作者可继续配置目标用户和发送策略。
- 有目标语言时，继续为当前临时消息创建外部机翻任务。
- 临时消息的审核结果只冻结在当前人工消息任务版本中，不创建或反向修改消息模板。
- 临时消息列表和任务详情继续展示多语言进度；单语言直写时展示原文审核结果，不展示机翻字段。

### 多语言进度面板

单语言直写模式展示：

- 生产方式：单语言直接编写；
- 审核语言；
- 审核组和专项审核标签；
- 当前状态和审核结论；
- “前往语言审核”入口。

不展示：

- 外部翻译任务编号；
- 机翻次数；
- 重试机翻；
- 机器翻译结果；
- 外部服务回调时间。

无需专项审核的单语言模板或临时消息直接展示“单语言内容，无需机器翻译，语言准备已完成”，并允许进入后续任务配置或业务审核。

### 多语言审核

需要专项审核的单语言原文继续进入独立“多语言审核”页签。审核抽屉根据生产模式调整：

- 标题显示“原文校对”。
- 左侧展示操作者提交的原文。
- 右侧展示可编辑的审核稿，初始内容与原文一致。
- 审核人可以修订并通过，或填写原因后驳回。
- 时间线显示“提交原文审核”，不显示“提交外部机翻”或“外部服务回调”。
- 继续执行提交人与审核人回避规则。

## 数据流

### 单语言且需要专项审核

```text
选择 ja-JP 为默认语言
  → 目标语言留空
  → 编写并保存日文内容
  → 提交语言审核
  → 创建原文审核批次
  → 日语专项审核组校对
  → 审核通过
  → 语言准备已完成
  → 进入业务审核
```

审核驳回后，消息模板或临时消息内容保持可修改。操作者修订内容并重新提交原文审核；旧审核稿和驳回原因保留在审计记录中。

### 单语言且不需要专项审核

```text
选择默认语言
  → 目标语言留空
  → 编写并保存内容
  → 语言准备自动完成
  → 进入业务审核
```

### 多语言模板

```text
选择默认语言及一个或多个目标语言
  → 提交外部机翻
  → 翻译返回待审核
  → 普通审核或专项审核
  → 所有目标语言通过
  → 进入业务审核
```

## 校验与异常处理

- 默认语言必填。
- 目标语言可为空，但不得与默认语言重复。
- 已选择正式渠道的必填内容必须完整；未选择渠道不参与校验。
- 默认语言内容修改后，尚未完成的原文审核失效，必须重新提交。
- 审核期间再次提交相同内容时使用内容哈希防止重复创建有效批次。
- 语言审核策略被停用或找不到审核组时，阻止提交并明确提示配置问题，不允许绕过审核。
- 原文审核不调用外部机翻接口，因此不存在机翻失败、机翻重试或回调超时。

## 审计要求

记录以下操作：

- 默认语言和目标语言变更；
- 单语言生产模式的确定；
- 原文审核提交、驳回、修订和通过；
- 审核人、审核时间、审核意见和内容版本；
- 审核策略变化导致的流程变化；
- 单语言模板或临时消息后续增加目标语言并发起机翻；
- 临时消息所属人工任务 ID 和任务内容版本。

## 验收标准

1. 操作者可以选择 `ja-JP` 作为默认语言并将目标语言留空。
2. 单语言日文模板或临时消息不会创建外部机翻任务，也不会显示外部任务编号或机翻结果。
3. 日语配置为专项审核语言时，提交按钮显示“提交语言审核”，对应模板版本或临时消息内容版本进入日语专项审核。
4. 审核抽屉展示日文原文和可编辑审核稿，不展示机器翻译相关字段和时间线。
5. 日语原文审核通过后，模板可以进入业务审核，临时消息可以继续完成人工任务配置与提交。
6. 不需要专项审核的单语言模板或临时消息可直接完成语言准备。
7. 目标语言不为空时，现有外部机翻和逐语言人工审核流程保持不变。
8. 修改默认语言时，目标语言列表自动移除与默认语言相同的语言。
9. 单语言模板后续增加目标语言时，可以当前默认语言内容正常发起机翻。
10. 站内信和 App Push 的内容编辑、校验与预览继续跟随正式渠道选择。
11. 临时消息可以单语言直接编写，并根据默认语言审核策略选择直接完成或进入专项语言审核。
12. 临时消息的原文审核结果只关联当前人工消息任务内容版本，不生成消息模板。

## 非本次范围

- 修改语言审核策略的配置规则。
- 接入新的翻译供应商。
- 自动判断原文语言。
- 允许未通过专项语言审核的模板或临时消息进入业务审核或发布。
