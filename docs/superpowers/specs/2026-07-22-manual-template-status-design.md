# 人工消息模板四状态设计

## 目标

人工消息模板仅保留以下四个业务状态：

- 草稿
- 审核中
- 驳回
- 已发布

“停用”不是人工消息模板能力，不提供停用入口，也不在列表、筛选或状态流转中出现。

## 适用范围

- 适用于人工消息模板模块中的人工专用模板和通用模板。
- 事件专用模板及事件自动化自身的状态模型不在本次范围内。
- 多语言任务仍保留自己的“无结果、翻译返回待审核、已通过”状态；它与模板主状态分层展示。

## 状态定义

### 草稿

模板尚未进入任何审核流程。新建模板、保存草稿、驳回后重新编辑并保存草稿，均为草稿。

### 审核中

模板已进入发布前审核链路，包括外部机翻、语言人工审核或业务审核中的任一阶段。具体进度继续由多语言流程和审核中心展示，模板列表不再细分“待业务审核”等中间状态。

### 驳回

业务审核未通过。模板允许再次编辑；编辑并保存后回到草稿，重新提交后进入审核中。

### 已发布

业务审核通过，模板可以被人工消息任务使用。人工消息模板发布后锁定，不允许直接编辑。

## 状态流转

```text
草稿
  ├─保存草稿────────────→ 草稿
  └─提交审核/提交机翻────→ 审核中

审核中
  ├─审核通过────────────→ 已发布
  └─审核驳回────────────→ 驳回

驳回
  ├─编辑并保存草稿──────→ 草稿
  └─修改后重新提交──────→ 审核中

已发布
  └─不可编辑，不存在停用操作
```

## 旧状态兼容

本地持久化数据加载时统一归一化：

| 旧状态 | 新状态 |
| --- | --- |
| 待业务审核 | 审核中 |
| 待审核 | 审核中 |
| 已驳回 | 驳回 |
| 已停用 | 草稿 |

这些旧状态仅用于数据迁移，页面和后续业务逻辑不得再次生成。

## 页面变化

- 人工消息模板状态筛选仅展示四个状态。
- 人工模板列表中不再出现“待业务审核、已驳回、已停用”等旧文案。
- 人工模板提交外部机翻、语言审核或业务审核后统一显示“审核中”。
- 审核中心驳回模板后显示“驳回”；通过后显示“已发布”。
- 已发布人工模板继续只读展示；草稿、审核中和驳回状态按现有权限规则处理。

## 数据与测试

- 为 `MessageTemplate` 增加受控的人工模板状态类型。
- 建立独立状态归一化函数，覆盖四个标准状态和全部旧状态映射。
- 在种子数据和本地存储迁移中应用归一化。
- 测试列表筛选只出现四状态、状态流程不再生成旧状态、旧数据可以安全迁移。

## 非目标

- 不新增模板停用、恢复或归档能力。
- 不修改事件通知规则的启用/停用状态。
- 不合并多语言状态与模板主状态。
