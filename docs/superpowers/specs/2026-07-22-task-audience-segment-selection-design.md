# 人工消息任务选择用户分群设计

## 目标

在人工消息任务的“目标用户”步骤中新增“用户分群”受众方式。操作者可以从“用户与受众”模块维护的已有分群中单选一个主分群，并可多选其他分群作为排除条件。

## 交互设计

“受众方式”新增“用户分群”选项，与“全部用户、指定用户、指定 VIP、指定代理、活动参与用户”并列。

选择“用户分群”后：

1. 展示必填的“用户分群”单选框。
2. 每个选项展示分群名称和当前人数，便于操作者确认规模。
3. 保留“排除分群”多选框，其数据同样来自“用户与受众”模块。
4. 已选主分群不能再次作为排除分群。
5. 切换主分群时，如果它已存在于排除分群中，自动从排除列表移除。

## 数据与状态

人工消息任务增加以下前端字段：

- `audienceType: "segment"`：标识受众来自用户分群。
- `audienceSegmentId`：主分群 ID。
- `excludedSegmentIds`：排除分群 ID 数组。

任务列表和详情仍使用分群名称作为面向操作者的受众摘要；分群 ID 用于编辑、复制和后续接口对接时恢复选择。

## 人数预览

- 原始受众人数使用主分群当前人数。
- 排除分群名称在预览区明确展示。
- 前端原型中的预计可发送人数仅作预估；最终人数仍以发送前的授权、退订、地区、抑制和重复 UID 校验结果为准。
- “刷新人数”沿用现有入口。

## 校验与异常处理

- 未选择主分群时不能进入下一步，提示“请选择用户分群”。
- 排除分群不能包含主分群。
- 如果复制或编辑的任务引用了已不存在的分群，保留受众摘要，同时要求操作者重新选择有效分群后才能继续。

## 范围

本次只修改人工消息任务前端：受众方式、选择控件、人数预览和任务数据保存。不修改用户分群的创建与编辑功能，也不新增后端接口。

## 验收标准

1. 人工消息任务可选择“用户分群”。
2. 主分群只能单选，并展示名称和人数。
3. 排除分群可以多选，且不能选择主分群。
4. 选择结果会写入任务数据，并可在编辑或复制时恢复。
5. 未选择主分群时无法进入下一步。
