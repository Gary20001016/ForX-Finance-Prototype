# 隐藏人工消息模板版本设计

## 目标

人工消息模板不再向操作者展示版本，减少“保存次数”和“发布版本”之间的理解歧义。系统继续保留内部版本快照，用于任务冻结、翻译批次关联和审计追溯。

## 展示范围

以下人工消息场景移除模板版本：

- 人工消息模板列表的“版本”列。
- 人工消息模板只读详情中的版本字段。
- 人工消息模板的多语言流程和使用任务标题。
- 新建人工消息任务选择模板时的版本文本。
- 人工消息任务列表、任务详情和任务摘要中的“模板版本”。
- 人工消息页面中“模板版本”相关说明文案。

操作者仍可通过模板名称和模板编号识别模板。

## 保留范围

- `MessageTemplate.version` 与 `MessageTask.templateVersion` 保留为内部字段。
- 翻译批次继续记录内部模板快照版本。
- 保存、审核、发布与历史任务关联逻辑不改变。
- 系统事件定义版本、事件通知内容版本及其展示不改变。
- 旧任务中已经包含版本的展示文本需要通过模板 ID 映射为模板名称；缺少模板 ID 时，回退为去除尾部版本号后的旧文本。

## 页面行为

- 模板选择器显示“模板名称 · 渠道”，不显示 `v1/v2`。
- 人工任务列表的模板列统一命名为“消息模板”，显示模板名称。
- 人工任务详情显示“消息模板”，不再显示模板版本。
- 模板列表抽屉标题仅显示模板名称和功能名称。
- 使用任务查询优先通过 `templateId` 关联，兼容旧的“模板编码 + 版本”字符串。

## 测试范围

- 人工模板列表、详情和抽屉不出现模板版本。
- 新建人工任务的模板选项不出现版本。
- 人工任务列表和详情显示模板名称，不显示版本。
- 模板使用任务仍能正确关联历史任务。
- 事件版本相关页面和字段保持不变。
