# 组合分群交集与并集设计

## 目标

让操作者在创建或编辑“组合分群”时，可以直观选择多个已有分群，并指定采用并集或交集。第一期聚焦前端交互和原型数据展示，不实现真实后端集合计算。

## 交互设计

- 仅当分群类型为“组合分群”时展示“组合规则”模块，不再展示动态条件的字段、运算符和阈值。
- 操作者必须选择至少两个已有分群。
- 集合运算提供两个互斥选项：
  - 并集：属于任意一个所选分群的用户均进入结果集。
  - 交集：必须同时属于全部所选分群的用户才进入结果集。
- 已选分群以可删除标签或多选框形式展示，并显示分群名称、当前人数和类型。
- 页面即时展示集合表达式，例如 `近30天沉默交易用户 ∪ ETH 活跃用户`。
- 页面展示原型预估人数：并集使用所选分群人数之和并做上限约束；交集使用所选分群最小人数的一定比例。该数字仅用于演示，并明确标记为预估。
- 保存后，列表“规则摘要”和详情页均展示集合表达式及运算类型。

## 数据模型

`AudienceSegment` 增加可选字段：

- `setOperation`: `union | intersection`
- `sourceSegmentIds`: 参与组合的分群 ID 数组

非组合分群不使用这两个字段。历史组合分群缺少字段时，编辑页提供默认来源分群，不影响现有列表展示。

## 校验与异常处理

- 组合分群少于两个来源分群时禁止保存，并提示“请至少选择 2 个分群”。
- 不允许选择正在编辑的分群自身，避免直接循环引用。
- 来源分群选择发生变化时，实时更新表达式和预估人数。
- 动态条件、静态名单保持现有行为，本次不扩展嵌套条件组或差集。

## 范围

本次只修改前端原型的创建/编辑弹窗、列表摘要和详情展示；不新增接口、不进行真实 UID 集合运算，也不实现多层嵌套组合。
