# 已发布人工模板演示数据设计

## 目标

在人工消息模板列表提供一条完整、可查看且不可编辑的已发布模板，用于直接验证“业务审核通过后锁定”的前端效果。

## 演示模板

- 模板 ID：`TPL-1007`
- 模板编码：`vip_benefit_update`
- 模板名称：`VIP 等级权益调整通知`
- 适用场景：`manual`
- 状态：`已发布`
- 版本：`v5`
- 多语言状态：`全部审核通过`
- 默认语言：`zh-CN`
- 目标语言：`en-US`
- 渠道：站内信、Push
- 风险等级：中

模板包含完整的 Web/App 共用站内信内容、App Push 内容、跳转链接、变量和所有者信息。模板列表应显示禁用的编辑入口，多语言流程应显示英文审核结果并保持只读。

## 翻译批次

新增与模板对应的 `MT-260716-007` 翻译批次和一个已通过的英文子任务。批次应包含源文案、机器翻译、人工修订结果、最终审核结果、审核人和审核时间，以支持多语言进度抽屉完整展示。

## 本地数据兼容

原型使用 localStorage 持久化状态。迁移旧状态时，模板和翻译批次需要按 ID 与最新种子数据合并：

- 已保存的同 ID 对象继续保留用户本地变更。
- 最新种子数据中本地不存在的模板和翻译批次自动追加。
- 不清空任务、审批、语言策略或其他现有本地状态。
- 合并过程按 ID 去重，重复刷新不得重复插入演示数据。

## 验收标准

1. 进入人工消息模板列表可看到 `VIP 等级权益调整通知`。
2. 模板状态为“已发布”，适用场景为“人工消息”。
3. 编辑按钮禁用，无法修改模板。
4. 多语言流程可查看 `en-US` 的机器翻译与最终审核结果。
5. 浏览器已有旧 localStorage 数据时无需手动清缓存也能看到该模板。
6. 页面刷新后不会重复生成模板或翻译批次。
