# 消息中心前端闭环补齐设计

## 目标

在不接入真实后端的前提下，把现有 Arco Design React 原型补齐为可完整演示的前端产品：任务、模板、外部机翻、人工翻译审核、业务审批、Web 站内信、App Push、发送记录、白名单、事件和分析之间共享同一份状态，并且所有关键按钮产生可见的数据变化。

## 架构

- 使用统一 `prototypeStore` 保存模板、翻译批次、任务、审批、用户消息、发送记录和跳转白名单。
- Store 使用浏览器本地存储持久化，并提供测试重置入口；页面只调用领域动作，不直接修改 Mock 数据。
- 外部机翻、APNs/FCM 和导出等外部能力由 Mock Adapter 模拟异步任务、回执与错误状态，接口签名可被真实 API 替换。
- 新建任务同时支持已审核模板和临时消息。临时消息与模板共用内容模型、语言生产和预览组件。

## 必须完成的交互

1. 模板可编辑 Web/Push 内容、变量、目标语言并创建首次机翻批次。
2. 翻译任务可重试、人工修订、通过或驳回，状态回写并控制发布门禁。
3. 任务支持模板/临时消息、五类受众、动态估算、样例用户、Web/Push 配置和双端预览。
4. 草稿、提交审核、复制、暂停、恢复、取消和审批均更新真实前端状态。
5. 审批展示所选对象的冻结内容、语言、受众、Web/Push 预览和差异；高风险确认及驳回原因强校验。
6. 用户消息已读状态跨路由持久化；业务动作按有效期和备案链接执行。
7. 发送记录筛选、Tab、Push 回执字段、临时错误重试和永久错误治理可操作。
8. 白名单支持新增、编辑、启停和有效期；事件支持详情、模板预览和测试事件。
9. 分析筛选生效，并展示 Web/Push 发送、送达、失败、重试、点击与成本对比。

## 验收

- 所有关键状态转换都有自动化交互测试。
- `npm run test:run` 与 `npm run build` 通过。
- 浏览器手工走通模板机翻、临时消息、审批和用户消息四条主流程。
