# 用户分群字段级编辑日志设计

## 目标

在用户分群详情中增加可追溯的字段级编辑日志，让操作者能够查看分群何时、由谁、修改了哪些内容以及修改前后的值。

## 页面设计

- 在分群详情抽屉的基础信息下方新增“编辑日志”模块。
- 日志按时间倒序排列，最新记录在最上方。
- 每条记录展示操作类型、操作人和操作时间。
- 每条记录下展示字段级变更明细，格式为“字段：旧值 → 新值”。
- 新建分群生成一条“创建分群”记录，展示首次写入的关键字段。
- 编辑已有分群并保存后，自动追加一条“编辑规则”记录。
- 没有字段变化时不生成新的编辑日志。

## 记录字段

日志比较以下字段：

- 分群名称
- 分群类型
- 业务用途
- 所有者团队
- 数据刷新
- 规则摘要
- 集合运算
- 来源分群

集合运算以“并集 / 交集”展示，来源分群以分群名称展示，不直接向操作者暴露内部数组格式。

## 数据模型

`AudienceSegment` 增加可选字段 `editLogs`。每条日志包含：

- `id`: 日志 ID
- `action`: `创建分群 | 编辑规则`
- `operator`: 操作人
- `operatedAt`: 操作时间
- `changes`: 字段变更数组

每条字段变更包含 `field`、`before` 和 `after`。首次创建时 `before` 使用“—”。

## 原型行为

- 当前操作者使用原型账号名 `Gary Ma`。
- 前端保存编辑时即时比较旧值与新值并写入本地页面状态。
- 现有模拟分群补充至少一条历史日志，确保打开详情即可查看效果。
- 本期不新增后端接口、不持久化到服务器，也不支持日志删除或修改。
