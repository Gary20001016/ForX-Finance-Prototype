# 一次性人工消息任务排期设计

## 结论

当前人工消息任务是一次性任务，仅支持立即发送和指定时间发送，不承担周期发送能力。

## 规则

- 立即发送：不展示计划发送时间和任务时区。
- 指定时间发送：同时展示计划发送时间和任务时区，两个字段均必填。
- 切回立即发送：清除计划发送时间和任务时区。
- 每个 UID 在本任务内固定发送一次，由受众快照、幂等键和后台执行保证。
- 不展示接收次数、频控时间范围、发送速率或 UID 去重开关。
- 若未来需要同一任务多次发送，应新增独立周期任务类型，配置开始时间、结束时间、周期和最大执行次数。

## 数据

- 立即发送保存为`立即`。
- 指定时间保存计划时间及任务时区。
- 不在一次性人工任务保存任务级`frequency_policy`。

## 兼容

- 历史任务记录保持原值。
- 事件自动化继续使用事件触发、TTL、幂等和事件级频控，不受本设计影响。
