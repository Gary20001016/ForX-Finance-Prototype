# 人工消息任务自定义频控设计

## 目标

明确区分“本任务每个 UID 仅发送一次”和“跨任务用户频控”，允许操作者自定义同类消息在滚动时间窗口内的最大接收次数。

## 交互

- 固定提示：本任务每个 UID 仅发送一次，失败重试不计为新的发送次数。
- 频控策略提供“不启用频控”和“自定义频控”。
- 不启用时不展示次数和时间窗口字段。
- 自定义时填写最大次数、窗口数值，并选择小时或天。
- 示例：`活动通知 24 小时内最多接收 3 次`。

## 数据

人工任务保存 `frequency_policy`：`enabled`、`max_count`、`window_value`、`window_unit`和固定范围`同消息分类`。次数和窗口必须为大于 0 的整数。

## 边界

- 频控用于受众执行时排除近期已达到同类消息上限的用户，不让单个任务重复发送。
- UID 去重和任务幂等始终生效，不受频控开关影响。
- 事件自动化的事件级频控保持不变。
