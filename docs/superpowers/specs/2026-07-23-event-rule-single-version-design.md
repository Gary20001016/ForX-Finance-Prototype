# 事件通知规则单版本设计

## 目标

事件通知规则不再维护规则内部的内容版本。每条规则只绑定一个已发布的事件消息模板，需要更新通知内容时新建另一条规则，并由操作者在规则名称中区分版本，例如“提现成功通知 V2”。

## 前端表现

- 删除规则列表的“当前内容版本”列。
- 删除规则详情中的“内容版本”列表、“创建内容版本”操作和内容版本编辑弹窗。
- 删除最近触发中的“冻结内容”列。
- 创建规则时继续从已发布的事件消息模板继承内容，不产生可继续编辑的内容版本入口。
- 规则详情展示绑定的消息模板，不展示规则内部内容版本。

## 提审与审核

- 提审可以直接进行，不要求必须选择旧规则。
- 提审弹窗提供“审核通过后暂停的规则（可选）”。
- 候选范围为除当前规则外的全部“已启用”规则，不限制系统事件。
- 提审只冻结待暂停规则列表，不立即改变任何旧规则状态。
- 审核通过时，在一次状态变更中启用新规则并暂停所选旧规则。
- 审核驳回或撤回时，不改变所选旧规则状态。
- 如果审核前某条待暂停规则已不再处于“已启用”，审核失败，避免部分执行。

## 数据兼容

本次以现有前端原型为范围。底层 `RuleContentVersion` 数据继续作为规则绑定模板内容的兼容快照，但不再向操作者暴露，也不再从规则页面创建新内容版本。

## 文案

- “替换旧规则”统一改为“审核通过后暂停规则”。
- 未选择暂停规则时，详情显示“审核通过后直接启用”。
- 已被新规则暂停的规则显示“已由 {规则编号} 暂停”。
