# 人工任务风险默认值与基础信息布局设计

## 目标

- 删除人工任务第一步中无实际配置能力的“发送方式”说明模块。
- 新建人工任务的风险等级默认改为“低”。
- 保证界面默认值、草稿数据、临时模板、预览和提交使用一致的风险兜底。

## 界面调整

- 删除“发送方式”标题。
- 删除“当前创建人工发送任务。系统事件自动通知请前往‘事件通知规则’配置。”提示。
- 删除该模块占用的分隔区域。
- “内容来源”直接排列在“发送渠道”之后。

## 风险等级规则

- 全新人工任务默认风险等级为“低”。
- 编辑已有任务时保留任务原风险等级。
- 复制已有任务时保留来源任务原风险等级。
- 操作者仍可在任务创建页主动选择“低 / 中 / 高 / 关键”。
- 表单值缺失时，草稿保存、临时模板、任务摘要和正式提交统一使用“低”，不得在不同环节回退为“中”。

## 非目标

- 不取消风险等级选择能力。
- 不改变消息分类默认性质。
- 不改变系统配置中的分类默认风险或历史任务数据。
- 不调整风险等级对应的审批规则。

## 验收

- 第一页不再展示“发送方式”说明模块。
- 新建空白任务显示风险等级“低”。
- 编辑和复制任务显示原任务风险等级。
- 缺省保存与提交结果的风险等级为“低”。
