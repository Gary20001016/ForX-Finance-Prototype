# 事件自动消息架构设计

## 1. 目标

拆除“系统事件 + 系统任务”的混合模型，将自动消息拆为事件目录、事件通知规则、规则内容版本、触发记录和渠道发送记录。人工消息任务继续作为一次性任务独立存在。

## 2. 对象模型

### 2.1 事件目录

事件目录只描述业务事实和数据协议，包括事件编码、名称、来源服务、业务线、Schema 版本、字段、健康状态和最近调用。事件定义不绑定模板、渠道或发送状态。

### 2.2 事件通知规则

事件通知规则是长期监听配置，包含关联事件、条件、事件主体映射、当前内容版本、渠道、去重键、频控、TTL 和重试。规则状态为：`草稿`、`待审核`、`待修改`、`已启用`、`已停用`、`已取消`、`已过期`。

### 2.3 规则内容版本

已启用规则不可直接覆盖当前内容。操作者创建新版本，完成外部机翻、人工审核和业务审核后发布；发布通过原子切换 `currentVersionId`，规则始终保持已启用。版本状态为：`草稿`、`机翻处理中`、`待人工审核`、`待审核`、`待生效`、`当前生效`、`已替换`、`已取消`、`已回滚`。

### 2.4 触发记录

每个事件实例匹配规则后生成独立触发记录，保存事件实例 ID、事件编码、规则 ID、规则版本、内容版本、幂等键、用户、接收时间、完成时间和结果。触发状态为：`已接收`、`已过滤`、`重复抑制`、`处理中`、`已完成`、`部分失败`、`失败`、`已过期`。

### 2.5 渠道发送记录

每条站内信或 Push 记录关联 `triggerId`。渠道记录继续管理提交、送达、点击、失败、重试、Token 失效和费用。

## 3. 关系

```text
事件目录 1 ── N 事件通知规则
事件通知规则 1 ── N 规则内容版本
事件通知规则 1 ── N 触发记录
触发记录 1 ── N 渠道发送记录
规则内容版本 N ── 1 消息模板版本
```

系统事件定义“发生了什么”；事件通知规则定义“应该如何通知”；消息模板定义“具体说什么”；触发记录说明“这次执行了什么”；渠道发送记录说明“是否送达”。

## 4. 信息架构

后台导航调整为：

- `人工消息任务`：只展示一次性人工发送任务。
- `事件通知规则`：管理长期自动通知规则及其内容版本。
- `消息模板`：管理内容和多语言生产。
- `事件目录`：管理事件定义和 Schema。
- `触发记录`：查看事件匹配、过滤、幂等和执行结果。
- `渠道发送记录`：查看站内信和 App Push 实际投递结果。

原 `消息任务` 页面不再展示事件触发任务；原 `系统事件` 页面不再展示“关联触发任务”，改为“关联通知规则”。

## 5. 核心流程

### 5.1 规则创建

```text
创建规则 → 草稿 → 提交审核 → 待审核
├──驳回审核 → 待修改 → 编辑规则 → 提交审核
└──通过审核 → 已启用
```

### 5.2 内容无中断替换

```text
规则保持已启用，当前版本 V1
└──创建内容版本 V2 → 草稿
    └──提交外部机翻 → 机翻处理中
        └──机翻完成 → 待人工审核
            └──人工审核通过 → 待审核
                └──业务审核通过 → 待生效
                    └──发布版本 → 原子切换
                        ├──V1 → 已替换
                        ├──V2 → 当前生效
                        └──规则 → 保持已启用
```

切换前已经接收的事件继续使用 V1；切换后接收的事件使用 V2。幂等键基于规则 ID 和事件实例 ID，不能包含内容版本，避免切换造成重复发送。

### 5.3 单次触发

```text
接收事件 → 匹配规则 → 条件校验 → 幂等校验 → 生成触发记录
→ 冻结规则和内容版本 → 生成渠道记录 → 汇总触发结果
```

单次触发成功或失败不改变规则的 `已启用` 状态。

## 6. 前端页面

### 6.1 人工消息任务

列表只读取 `triggerType=manual` 的任务；标题、描述和新建按钮均使用“人工消息任务”。人工任务创建页不再提供“系统事件触发”入口。

### 6.2 事件通知规则

列表展示规则、事件、条件、当前内容版本、渠道、状态、近 24 小时触发量、成功率和负责人。详情抽屉展示事件主体映射、去重、TTL、重试、版本历史和最近触发记录。页面支持创建规则、停用、启用、创建内容版本、推进机翻和审核、发布版本、取消版本及回滚。

### 6.3 事件目录

列表展示事件编码、业务线、Schema 版本、来源服务、字段数、关联规则数、调用量、失败率、健康状态和最近调用。详情展示字段 Schema、关联规则和测试事件入口。

### 6.4 触发记录

列表支持按事件、规则、状态、时间和关键词筛选。详情展示接收、匹配、幂等、版本冻结、渠道汇总和失败原因。

### 6.5 渠道发送记录

页面标题改为“渠道发送记录”，增加触发记录 ID 列和筛选，详情可跳转或关联查看对应触发记录。

## 7. 原型数据与兼容

前端原型新增 `rules`、`ruleVersions` 和 `triggerRecords` 三组状态。现有事件触发 `MessageTask` 在初始化或迁移时转换为规则，人工任务列表不再读取它们。测试事件通过已启用规则生成触发记录和渠道发送记录。

## 8. 验收标准

1. 导航和路由完整区分人工任务、事件通知规则、事件目录、触发记录和渠道发送记录。
2. 人工任务列表不再出现系统事件触发任务。
3. 一个事件可展示多个关联通知规则。
4. 已启用规则可创建、审核并发布新内容版本，发布前旧版本继续生效，发布后原子切换。
5. 单次测试事件生成触发记录和渠道发送记录，但规则保持已启用。
6. 触发记录和渠道记录可互相追溯，并显示实际使用的规则版本与内容版本。
7. 所有新增按钮、菜单、筛选、详情和状态流转均可操作且有自动化测试。
