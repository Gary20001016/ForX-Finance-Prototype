# 系统事件、消息任务与消息模板整合设计

**日期：** 2026-07-13

## 目标

把当前“系统事件直接绑定模板”的弱关联改成完整的三层模型：系统事件定义事实，事件触发任务定义发送策略，已发布模板版本定义最终内容。人工群发任务继续支持选择模板或临时消息，不强制绑定系统事件。

## 关系模型

```text
系统事件 1 ── N 事件触发任务 N ── 1 已发布模板版本
人工发送任务 ── 1 已发布模板版本或临时消息
```

- 系统事件只维护事件编码、版本、调用方、变量结构和运行状态。
- 事件触发任务维护触发条件、模板版本、变量映射、渠道、优先级、去重、有效期和重试策略。
- 消息模板维护 Web 站内信、App Push、多语言、变量和不可变发布版本。
- 同一事件可建立多个任务，例如对普通提现失败和高风险提现失败使用不同模板与优先级。
- 同一已发布模板可被多个任务复用，但模板变量必须全部能从事件变量映射得到。

## 数据模型

### 消息任务

`MessageTask` 新增：

- `triggerType`: `manual | event`。
- `templateId`、`templateVersion`：冻结实际使用的模板版本。
- `eventConfig`：仅系统事件任务存在。

`eventConfig` 字段：

- `eventId`：系统事件编码。
- `eventVersion`：事件版本。
- `conditionExpression`：触发表达式，空值表示事件到达即触发。
- `variableMappings`：事件字段到模板变量的显式映射。
- `dedupeKey`：幂等键表达式。
- `eventTtlSeconds`：事件发生后允许生成消息的时长。
- `maxRetries`：临时失败最大重试次数。
- `retryBackoffSeconds`：首次退避秒数。

### 系统事件

`SystemEventDefinition` 不再保存 `template` 和 `templateId`。系统事件详情通过 `eventId` 反查关联任务，并展示每个任务使用的模板版本、状态、条件和渠道。

### 模板

`MessageTemplate` 不再用 `eventCode` 表示单一直接绑定。模板列表通过 `templateId` 反查使用它的人工任务和事件任务。事件任务只能选择“全部目标语言审核通过且业务审批发布”的模板版本。

### 审批与发送记录

- 审批快照增加触发方式和 `eventConfig`，确保审核人看到事件、条件、映射、去重和重试策略。
- 人工任务审批通过后进入“待发送”。
- 事件任务审批通过后进入“已启用”，等待事件调用。
- 测试事件根据事件编码找到已启用任务，再使用任务冻结的模板生成 Web/Push 测试发送记录。
- 没有已启用任务或变量映射不完整时，测试事件必须明确阻断并显示原因。

## 页面设计

### 新建消息任务

第一步增加“触发方式”：

- 人工发送：保持现有模板/临时消息、五类受众和发送时间配置。
- 系统事件：必须选择系统事件和已发布模板；显示事件版本、调用方、状态和变量；配置条件表达式、变量映射、去重键、事件有效期及重试策略。

系统事件任务的目标用户固定为“事件主体用户”，不展示全站、VIP、代理或活动参与用户选择。发送策略保留 Web/Push、消息有效期和优先级，但不展示定时发送选项。

### 消息任务列表与详情

- 类型列明确显示“人工发送”或“系统事件触发”。
- 事件任务显示事件编码。
- 详情抽屉展示事件版本、条件、变量映射、去重键、事件有效期和重试策略。
- 除“发送中”和“已完成”外仍允许再次编辑；重新编辑会撤回旧审批。

### 系统事件

- 列表把“绑定模板”替换为“关联触发任务”。
- 详情展示变量结构与关联任务。
- “创建触发任务”跳转任务创建页并预选当前事件。
- 注册事件时不再选择模板。
- “发送测试事件”只有在存在已启用任务且映射完整时才生成发送记录。

### 消息模板

- 列表把“事件编码”替换为“使用任务”。
- 模板详情可看到使用该版本的人工任务和事件任务。
- 模板编辑和多语言发布流程不直接选择系统事件。

## 校验规则

- 系统事件任务必须选择事件和已发布模板。
- 模板中的每个变量都必须有且仅有一个事件字段映射。
- 映射来源必须存在于所选事件变量列表。
- 去重键不能为空，默认使用事件编码、事件主体 UID 和事件唯一 ID。
- 事件有效期必须大于 0，最大 604800 秒。
- 最大重试次数为 0–10，首次退避为 1–3600 秒。
- 风控事件默认风险等级为“关键”，Push 优先级为“紧急”，但操作者仍可在审批前调整。
- 未发布模板、翻译审核未全部通过的模板禁止用于事件任务。

## 前端状态流

1. 操作者注册或选择系统事件。
2. 从系统事件详情创建触发任务，或在任务创建页选择系统事件触发。
3. 选择已发布模板并完成变量映射、条件和发送策略。
4. 提交审批，冻结事件、模板版本和策略。
5. 审批通过后任务变为“已启用”。
6. 测试事件或真实接口事件匹配任务，渲染模板并生成发送记录。
7. 发送记录与数据分析按事件编码、任务和模板版本追踪。

## 测试范围

- 类型与状态层：事件任务保存字段、旧审批撤回、审批通过变为已启用。
- 创建页：触发方式切换、事件预选、模板发布门禁、变量映射和必填校验。
- 系统事件页：关联任务展示、创建任务跳转、无启用任务阻断、测试发送成功。
- 任务列表：事件编码和事件策略详情展示，再次编辑保留事件配置。
- 模板列表：使用任务数量取代单一事件编码。
- 回归：人工任务、临时消息、多语言、审批、用户消息中心、Web/Push 发送记录不受影响。

## 范围边界

- 本次只开发前端和本地状态模拟，不调用真实事件总线、外部机翻或 APNs/FCM。
- 不增加复杂可视化规则编排器；触发条件使用可审计的单行表达式。
- 不支持一个任务同时监听多个事件；需要多个事件时分别创建任务。
