## 统一产品边界

- 当前仓库交付前端交互原型，使用模拟数据，不连接生产接口。
- 产品一期必须正式交付 Web / App 共用的站内信和独立的 App Push；站内信两端共用消息 ID、未读数与已读状态，两种消息渠道均纳入生产验收。
- 邮件、短信、自动化旅程和高级营销归因不在产品一期范围。
- 系统事件不直接绑定模板。事件由已启用的事件触发任务选择已发布模板版本。

## 变更规则

- 修改详细功能时，只更新归属模块，并检查引用该模块的总览或其他模块。
- 修改跨模块流程、产品范围、版本或关键指标时，同时更新 [消息中心总览](./00-消息中心总览.md)。
- 新增字段必须说明类型、必填性、来源、校验和脱敏要求。
- 新增状态必须同步状态流、允许操作、异常处理和验收标准。
