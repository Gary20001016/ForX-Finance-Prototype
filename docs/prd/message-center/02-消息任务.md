# 人工消息任务 PRD

## 1. 模块摘要

人工消息任务用于一次性、由操作者发起的消息发送。它承载内容、人工受众、发送时间、审批和执行结果，不再承载系统事件自动通知；事件自动通知由[事件自动化](./04-系统事件.md)独立管理。

## 2. 目标与范围

- 支持新建、保存草稿、编辑任务、复制任务、提交/撤回审核、取消、暂停、恢复、重试失败项和查看详情。
- 支持模板消息和临时消息；临时消息选择多语言时进入外部机翻与人工审核流程。
- 支持全站、指定 UID、VIP、代理和活动参与用户五类人工受众。
- 冻结通过审批的内容版本、受众快照、渠道、时间和有效期，保证发送可追溯。

## 3. 用户与使用场景

| 角色 | 场景 |
|---|---|
| 运营人员 | 创建一次性运营、公告、资产或奖励消息 |
| 风控人员 | 创建一次性高风险强提示 |
| 审核人员 | 检查冻结内容、受众、渠道和计划时间 |
| 任务所有者 | 编辑可编辑任务，或复制历史任务创建新草稿 |

## 4. 前置条件与依赖

- 模板与临时多语言内容遵循[消息模板与多语言](./03-消息模板与多语言.md)门禁。
- 目标用户由[用户与受众](./05-用户与受众.md)生成不可变快照。
- 提交、审批和发布由[审核与发布](./06-审核与发布.md)控制。
- 用户级和渠道级结果进入[渠道与发送记录](./07-渠道与发送记录.md)。

## 5. 主流程

```text
草稿
└─提交审核→ 待审核
             ├─驳回审核→ 待修改
             │             └─编辑任务并重新提交→ 待审核
             └─通过审核
                   ├─指定时间发送→ 待发送
                   │             └─系统启动发送→ 发送中
                   └─立即发送→ 发送中
                                  ├─暂停发送→ 已暂停
                                  │             └─恢复发送→ 发送中
                                  └─系统完成发送→ 已完成
```

## 6. 功能需求

### 6.1 创建入口

- 新建人工消息任务：创建空白草稿。
- 编辑任务：打开`草稿`、`待修改`或`待发送`任务；编辑`待发送`会使原审批失效并回到草稿。
- 复制任务：复制内容、受众条件和策略为新草稿，不复制任务 ID、审批、受众快照和发送结果。
- 模板消息：引用已发布模板版本。
- 模板候选仅包含`usage_scope=manual`或`shared`且全部目标语言审核通过的版本；事件专用模板不得出现在人工任务中。
- 临时消息：维护源文案；多语言内容先完成外部机翻、人工审核和业务审批。
- 临时消息创建机翻后立即展示批次、逐语言状态、完成比例和当前阶段；普通语言在当前页面校对，专项语言跳转“多语言审核”。

### 6.2 配置步骤

1. 内容与多语言：选择模板或临时内容，完成目标语言生产与审核。
2. 目标用户：配置五类人工受众；指定 UID 支持手动输入和 CSV 合并去重。
3. 发送策略：选择站内信、App Push、立即发送/指定时间发送、时区、有效期、频控和优先级；仅选择指定时间发送时显示并要求填写计划发送时间。
4. 预览并提交：展示 Web 站内信、App 站内信和 App Push 的真实预览及完整摘要。

### 6.3 编辑规则

- `草稿`、`待修改`、`待发送`可以直接执行“编辑任务”。
- `待审核`必须先执行“撤回审核”，回到`草稿`后再编辑。
- `发送中`、`已暂停`、`已完成`、`已取消`、`已过期`不可编辑；需要复用时执行“复制任务”。
- 已发布模板版本不可在任务内覆盖，只能切换到另一个已发布版本。

### 6.4 指定 UID

- 支持手动输入和 UTF-8 CSV 导入，两种来源合并去重。
- CSV 必须包含`uid`列，单文件最大 10 MB、最多 100,000 行。
- 展示有效、重复、无效、排除和最终 UID 数量；有无效项时必须确认排除结果后才能继续。

### 6.5 幂等

- 保存草稿：`task_id + draft_version`，同版本重复保存只更新一次。
- 提交审核：`task_id + submitted_version`，重复点击不创建重复审批。
- 启动发送：`task_id + publish_version`，重复调度不重复创建用户消息。
- 用户消息：`task_id + publish_version + uid`，同一发布版本对同一用户只创建一次。
- 渠道发送：`message_id + channel + destination_id`，供应商超时查询原结果后再决定是否重试。

## 7. 字段定义

| 分组 | 字段 | 说明 |
|---|---|---|
| 标识 | `task_id`、`task_name` | 唯一任务编号和名称 |
| 基础 | `category_code`、`risk_level`、`reason`、`ticket_id` | 分类、风险和业务依据 |
| 内容 | `content_source`、`template_id`、`template_version`、`locales` | 模板或临时内容及冻结版本 |
| 受众 | `audience_type`、`audience_config`、`audience_snapshot_id` | 人工受众配置与不可变快照 |
| UID 导入 | `uid_audience_snapshot` | 文件名、有效、无效、重复、排除和最终 UID 数 |
| 渠道 | `channels`、`fallback_policy` | `inbox`、`push`及兜底策略 |
| 时间 | `schedule_type`、`schedule_at`、`timezone` | 仅支持立即发送或指定时间发送；指定时间模式必须填写`schedule_at` |
| 生命周期 | `expire_at`、`retention_days` | 消息有效期和保留时间 |
| 控制 | `frequency_cap`、`priority`、`max_retries` | 频控、优先级和执行重试 |
| 任务状态 | `task_status` | 九种任务生命周期状态 |
| 审批 | `approval_status`、`submitted_version` | 与任务状态独立 |
| 发送结果 | `delivery_result` | 与任务状态独立 |
| 多语言 | `translation_batch_id`、`multilingual_stage`、`unfinished_locales` | 翻译批次、当前阶段和未完成语言 |
| 审计 | `created_by`、`created_at`、`updated_by`、`updated_at` | 创建和修改信息 |

## 8. 状态、操作与影响

任务状态固定为：`草稿`、`待审核`、`待修改`、`待发送`、`发送中`、`已暂停`、`已完成`、`已取消`、`已过期`。

| 当前任务状态 | 可执行操作 | 操作后的任务状态 |
|---|---|---|
| 草稿 | 查看详情、编辑任务、复制任务、提交审核、取消任务 | 提交审核→待审核；取消任务→已取消；复制任务→新草稿 |
| 待审核 | 查看详情、复制任务、撤回审核、取消任务 | 撤回审核→草稿；审核通过→待发送或发送中；驳回审核→待修改；取消任务→已取消 |
| 待修改 | 查看详情、编辑任务、复制任务、提交审核、取消任务 | 提交审核→待审核；取消任务→已取消 |
| 待发送 | 查看详情、编辑任务、复制任务、取消任务 | 编辑保存→草稿且原审批失效；到时系统启动→发送中；取消任务→已取消 |
| 发送中 | 查看详情、复制任务、暂停发送、取消任务 | 暂停发送→已暂停；取消任务→已取消；系统完成→已完成 |
| 已暂停 | 查看详情、复制任务、恢复发送、取消任务 | 恢复发送→发送中；取消任务→已取消 |
| 已完成 | 查看详情、复制任务；部分失败/失败时可重试失败项 | 重试失败项→创建新草稿；原任务保持已完成 |
| 已取消 | 查看详情、复制任务 | 复制任务→新草稿；原任务不变 |
| 已过期 | 查看详情、复制任务 | 复制任务→新草稿；原任务不变 |

审批状态独立使用`未提交`、`审核中`、`通过`、`驳回`、`已撤回`、`已终止`。发送结果独立使用`未开始`、`处理中`、`成功`、`部分失败`、`失败`。任务执行结束后统一进入`已完成`，成功程度由发送结果表达。

## 9. 权限与审计

- 创建人与最终审核人不能相同。
- 保存、编辑、复制、提交、撤回、取消、暂停、恢复和重试均记录操作者、对象版本、时间和结果。
- 全站、紧急和高影响任务需要更高权限与升级审批。

## 10. 异常与边界

- 定时时间已过去、有效期早于发送时间、最终受众为 0：禁止提交。
- 模板或翻译未通过门禁：禁止提交。
- 指定 UID CSV 未确认排除结果：禁止进入发送策略。
- 重复点击提交、调度或重试：按幂等键返回原结果。

## 11. 数据与埋点

记录创建、保存、预览、提交、撤回、复制、审核、开始发送、暂停、恢复、取消、完成和重试。分析维度包括分类、风险、内容来源、模板版本、受众类型、渠道和创建团队。

## 12. 验收标准

1. 人工任务列表不出现事件通知规则或事件触发执行。
2. 支持模板消息、临时消息和临时多语言完整流程。
3. 指定 UID 支持手动输入与 CSV 合并去重及确认门禁。
4. 页面同时展示 Web 站内信、App 站内信和 App Push 预览。
5. 九种任务状态、审批状态和发送结果分别展示，操作后状态变化符合本表。
6. 重复操作不会重复创建审批、用户消息或渠道发送实例。
7. 列表分别展示“多语言流程”和“发送进度”，两者不得共用同一进度列。

## 13. 非本模块范围

系统事件自动通知、自动化旅程、A/B 实验和复杂批次编排不属于本模块。
